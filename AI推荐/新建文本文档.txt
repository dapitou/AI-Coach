# 运动内容数字化标准属性清单 (七层架构版)

================================================================
一、 核心层级穿透图 (ER 拓扑)
================================================================

[第1层：计划 (Plan)]  -> 定义“要达成什么结果” (宏观目标)
       │
       └── 1:N ── [第2层：阶段 (Phase)] -> 定义“每个月怎么变策略” (周期进阶)
                     │
                     └── 1:N ── [第3层：单周模板 (Weekly_Template)] -> 定义“每周怎么排课” (微观循环)
                                   │
                                   └── 1:N ── [第4层：课程 (Course)] -> 定义“今天练什么” (单次训练单元)
                                                 │
                                                 └── 1:N ── [第5层：环节 (Section)] -> 定义“动作怎么组合” (训练逻辑)
                                                               │
                                                               └── 1:N ── [第6层：动作实例 (Instance)] -> 定义“具体做多少” (执行指令)
                                                                             │
                                                                             └── 引用 ──> [第7层：动作库 (Library)] (静态元数据)

================================================================
二、 详细属性清单 (全中文定义)
================================================================

----------------------------------------------------------------
1. 计划层 (Plan) - 产品容器
   > 作用：用户的购买/订阅对象，定义整体方向。
----------------------------------------------------------------
| 属性名称 | 必填 | 说明与逻辑 |
| :--- | :--- | :--- |
| 计划唯一ID | 是 | 全局唯一标识 |
| 计划名称 | 是 | 展示给用户的标题 (如"12周力量举基础") |
| 核心目标分类 | 是 | **算法策略依据** (增肌/减脂/心肺/矫正/运动表现) |
| 适用等级 | 是 | 准入门槛 (初级/中级/高级) |
| 总周期长度 | 是 | 单位：周 |
| 每周训练频次 | 是 | 每周建议练几天 (如 4天) |
| 器械要求集合 | 是 | 自动聚合下层所有动作的器械 (如 "哑铃, 瑜伽垫") |
| 智能调整开关 | 是 | 是否允许AI根据用户疲劳度动态修改下层动作 |

----------------------------------------------------------------
2. 阶段层 (Phase) - 周期策略
   > 作用：解决训练的枯燥感和适应性，实现波浪式进阶。
----------------------------------------------------------------
| 属性名称 | 必填 | 说明与逻辑 |
| :--- | :--- | :--- |
| 阶段唯一ID | 是 | 全局唯一标识 |
| 归属计划ID | 是 | **[向上穿透]** 指向所属计划 |
| 阶段名称 | 是 | (如 "基础适应期", "大重量冲击期", "减负周") |
| 排序序号 | 是 | 在计划中的执行顺序 (1, 2, 3...) |
| 持续周数 | 是 | 该阶段维持几周 (如 4周) |
| 策略重心 | 是 | **疲劳算法参数** (容量累积/强度突破/主动恢复) |
| 包含的周模板ID | 是 | **[向下穿透]** 指向单周模板列表 |

----------------------------------------------------------------
3. 单周模板层 (Weekly Template) - 排课逻辑 [新增]
   > 作用：定义一周 7 天的训练节奏 (推-拉-腿-休-全身-休-休)。
----------------------------------------------------------------
| 属性名称 | 必填 | 说明与逻辑 |
| :--- | :--- | :--- |
| 模板唯一ID | 是 | 全局唯一标识 |
| 归属阶段ID | 是 | **[向上穿透]** 指向所属阶段 |
| 适用周次 | 是 | 该模板应用于阶段内的第几周 (如 "第1-3周") |
| 排课日程表 | 是 | **核心逻辑对象**，结构如下： |
| - 周一配置 | 是 | 类型(训练/休息) + 关联课程ID |
| - 周二配置 | 是 | 类型(训练/休息) + 关联课程ID |
| ... | ... | ... |
| - 周日配置 | 是 | 类型(训练/休息) + 关联课程ID |

----------------------------------------------------------------
4. 课程层 (Course) - 单次训练单元
   > 作用：用户点击“开始”后加载的实体。
----------------------------------------------------------------
| 属性名称 | 必填 | 说明与逻辑 |
| :--- | :--- | :--- |
| 课程唯一ID | 是 | 全局唯一标识 |
| 课程名称 | 是 | (如 "胸部力量基础A") |
| 课程类型 | 是 | 决定UI播放器模式 (跟练模式/视频流模式/音频模式) |
| 预估时长 | 是 | 单位：秒 (由下层动作时长累加) |
| 预估消耗 | 是 | 单位：千卡 (由下层MET值累加) |
| 预估疲劳负荷 | 是 | **疲劳算法输入** (基于容量模型预计算的疲劳分) |
| 包含环节列表 | 是 | **[向下穿透]** 有序的环节ID列表 |

----------------------------------------------------------------
5. 环节层 (Section) - 逻辑容器
   > 作用：定义具体的训练法 (超级组/EMOM/Tabata/循环)。
----------------------------------------------------------------
| 属性名称 | 必填 | 说明与逻辑 |
| :--- | :--- | :--- |
| 环节唯一ID | 是 | 全局唯一标识 |
| 归属课程ID | 是 | **[向上穿透]** 指向所属课程 |
| 功能类型 | 是 | **推荐逻辑依据** (热身/主训/激活/放松/技能教学) |
| 组合逻辑模式 | 是 | **播放逻辑依据** (直组/超级组/循环组/EMOM/AMRAP) |
| 循环配置 | 是 | 大循环次数 (Rounds) + 轮间休息时间 |
| 计时器配置 | 否 | 特殊计时规则 (如 EMOM 的 "每60秒一轮") |
| 包含动作列表 | 是 | **[向下穿透]** 有序的动作实例ID列表 |

----------------------------------------------------------------
6. 动作实例层 (Action Instance) - 执行指令 [核心泛化层]
   > 作用：将“死”的动作库变成“活”的训练指令。兼容举铁、跑步、静态支撑。
----------------------------------------------------------------
| 属性名称 | 必填 | 说明与逻辑 |
| :--- | :--- | :--- |
| 实例唯一ID | 是 | 全局唯一标识 |
| 引用动作ID | 是 | **[数据指针]** 指向静态动作库 (获取动作视频/肌肉属性) |
| 单双边模式 | 是 | 执行策略 (双边/左侧/右侧/交替) |
| 目标计量单位 | 是 | **练多少** (次数 / 时间-秒 / 距离-米) |
| 目标计量数值 | 是 | (如 12 / 60 / 5000) |
| 负荷类型 | 是 | **强度监控** (重量 / 自重 / RPE / 心率区间 / 配速) |
| 负荷数值 | 是 | (如 "20kg" / "RPE 8" / "Zone 2" / "5'30") |
| 节奏控制 | 否 | (如 "3-1-1-0" 控制离心与向心速度) |
| 组间休息 | 是 | 该动作完成后的休息时长 (秒) |
| 覆盖提示语 | 否 | 针对该课程的特殊指导 (覆盖库里的默认提示) |

----------------------------------------------------------------
7. 动作库层 (Library) - 静态元数据
   > 作用：底层的物理与生理字典。
----------------------------------------------------------------
| 属性名称 | 必填 | 说明与逻辑 |
| :--- | :--- | :--- |
| 动作唯一ID | 是 | 全局唯一标识 |
| 动作名称 | 是 | (如 "杠铃颈后深蹲") |
| 动作模式 | 是 | **动作替换逻辑依据** (蹲/铰链/推/拉/旋转/抗旋...) |
| 主动肌群 | 是 | **疲劳计算权重100%** (如 股四头肌) |
| 辅助肌群 | 否 | **疲劳计算权重50%** (如 臀大肌, 核心, 竖脊肌) |
| 代谢当量(MET) | 是 | **热量与虚拟容量计算依据** |
| 冲击等级 | 是 | **伤病规避依据** (无冲击/低冲击/高冲击) |
| 媒体资源包 | 是 | 视频/音频/封面图 |
| 适用场景标签 | 是 | **多态性筛选** (热身/主训/拉伸/测试) |

================================================================
三、 全链路逻辑穿透演示 (The Thread)
================================================================

**场景：用户正在执行“哑铃卧推”这个动作，系统在后台在思考什么？**

1. **[执行层] (动作实例)**
   - 系统读取指令：负荷="20kg", 次数="10次"。
   - **穿透逻辑**：通过 `引用动作ID` 抓取动作库数据。

2. **[生理层] (动作库)**
   - 系统识别：这是一个 "水平推" 模式，主动肌是 "胸大肌中束"，辅助肌是 "肱三头肌"。
   - **穿透逻辑**：将 容量(20kg*10) × 敏感度 传入疲劳算法，更新用户的【肌肉状态表】。

3. **[逻辑层] (环节)**
   - 系统判断：当前处于 "主训环节"，模式为 "超级组"。
   - **穿透逻辑**：做完这个动作**不休息**，立即跳转列表中的下一个动作（因为是超级组）。

4. **[日程层] (单周模板)**
   - 系统确认：今天是 "第1周 - 周一"。
   - **穿透逻辑**：完成该课程后，将周一的日程标记为【已完成】，解锁周二的课程预览。

5. **[策略层] (阶段)**
   - 系统定位：当前处于 "基础适应期" (Phase 1)。
   - **穿透逻辑**：因为是适应期，系统在后台记录 RPE，如果用户反馈太轻松，下周自动提升负荷（渐进超负荷）。

6. **[目标层] (计划)**
   - 系统校验：整个计划的目标是 "极致增肌"。
   - **穿透逻辑**：确保所有动作的节奏（Tempo）和组间休息都符合肌肥大原则（如控速、休息60-90秒）。
