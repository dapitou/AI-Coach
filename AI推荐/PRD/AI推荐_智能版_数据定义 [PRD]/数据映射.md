# 数据映射
> > *业务导读：系统的“翻译字典”，定义了不同维度数据（如肌肉和部位）之间的对应关系。*
> 定义：系统中各维度数据之间的逻辑关联与转换规则。
> 用途：为算法逻辑、数据校验、前端展示提供统一的标准参照。

## 解剖学层级映射
> 用途：定义“区域-部位-肌肉”的包含关系，用于筛选与疲劳聚合。
> > *业务导读：定义身体部位的从属关系，比如胸大肌属于胸部，胸部属于上肢。*
> 引用：动作属性 - 主动肌/辅助肌/稳定肌

### 映射表
| 区域 | 部位 | 包含肌肉 |
| :--- | :--- | :--- |
| **上肢** | **胸部** | 胸大肌上束、胸大肌中束、胸大肌下束、胸大肌内侧 |
| **上肢** | **背部** | 上背、中背、下背 |
| **上肢** | **肩部** | 三角肌前束、三角肌中束、三角肌后束、肩袖 |
| **上肢** | **手臂** | 肱二头肌、肱三头肌 |
| **下肢** | **臀部** | 臀大肌、臀中肌 |
| **下肢** | **腿部** | 股四头肌、腘绳肌、内收肌、腓肠肌 |
| **核心** | **核心** | 腹直肌、腹斜肌 |
| **全身** | **全身** | 涉及上述多个部位的组合 |

## 动作模式-热身功能映射
> 用途：基于主训模式推荐精准的热身动作。
> > *业务导读：根据主训练什么（比如推胸），决定热身该做什么（比如肩部激活）。*
> 引用：动作属性 - 动作模式 vs 动作功能(主要部位)

### 映射表
| 主训模式 (可多选) | 推荐热身策略 (功能 + 部位) |
| :--- | :--- |
| **水平推、垂直推** | `柔韧`+`肩部`、`柔韧`+`胸部`、`激活`+`肩部`、`协调`+`背部` |
| **水平拉、垂直拉** | `柔韧`+`背部`、`柔韧`+`肩部`、`激活`+`背部`、`协调`+`背部` |
| **膝主导、髋主导** | `柔韧`+`臀部`、`柔韧`+`腿部`、`激活`+`臀部`、`协调`+`核心` |
| **弓步、步态** | `平衡`+`腿部`、`激活`+`腿部`、`柔韧`+`臀部` |
| **旋转、核心稳定、脊柱屈伸** | `激活`+`核心`、`柔韧`+`背部`、`协调`+`全身` |

## 动作模式与主练部位映射
> 用途：用于校验动作定义的准确性，以及基于模式的推荐兜底。
> > *业务导读：定义动作模式（如水平推）通常练哪里（如胸部），用于数据校验。*
> 引用：动作属性 - 动作模式 vs 主要部位 [次要部位]

### 映射表
| 模式名称 | 主要部位 | 次要部位 |
| :--- | :--- | :--- |
| **水平推** | 胸部 | 肩部、手臂 |
| **垂直推** | 肩部 | 胸部、手臂 |
| **水平拉** | 背部 | 肩部、手臂 |
| **垂直拉** | 背部 | 手臂 |
| **膝主导** | 腿部 | 臀部 |
| **髋主导** | 臀部、腿部 | 背部 |
| **弓步** | 腿部、臀部 | - |
| **旋转** | 核心 | - |
| **核心稳定** | 核心 | 臀部 |
| **脊柱屈伸** | 核心 | 背部 |
| **步态** | 腿部、臀部 | 核心 |

## 4. 强度指标换算映射
> 用途：将不同的强度指标统一换算，用于负荷管理。
> > *业务导读：把“L1-L5”的难度等级翻译成具体的重量、心率和休息时间。*
> 引用：环节属性 - 训练强度/目标RPE

### 映射表
| 等级 | 力量负荷 | 自重次数基准 | 有氧心率(MHR) | 推荐组间休息 | 计时时长 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **L1 (低强度)** | < 60% 1RM | **8-10** | 50-60% | 30-60s | 30s |
| **L2 (中低强度)** | 60-70% 1RM | **10-12** | 60-70% | 60-90s | 30s |
| **L3 (中等强度)** | 70-80% 1RM | **12-15** | 70-80% | 90-120s | 45s |
| **L4 (高强度)** | 80-90% 1RM | **15-20** | 80-90% | 2-3min | 45s |
| **L5 (极限强度)** | > 90% 1RM | **20+ (力竭)** | > 90% | 3-5min | 60s |
