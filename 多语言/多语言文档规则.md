# 软件多语言文档规则

## 核心目标
基于软件页面截图（多页，含英文文本），按指定格式整理多语言初始文档，输出可直接交付翻译的 Excel 表格，确保不同 AI 执行时结果一致性。

## 一、字段格式
> 共6列
| 字段名 | 填写规则 |
|-------|----------|
| Key | 3级结构：范围_分类_内容（如common/页面缩写_固定分类_英文原词/缩写） |
| Description | 3级中文：范围/模块_分类_具体内容（与Key一一对应） |
| Char Check | 核心词汇填字符上限（如10/20），通用词汇可空 |
| zh_CN | 简体中文，截图英文准确翻译，软件界面风格 |
| zh_HK | 香港繁体中文 |
| en_US | 严格提取截图英文原文 |

## Key 命名规则
### 核心原则
简洁唯一、见名知义，下划线 “_” 分隔层级，无冗余。
### 命名格式
`范围_分类_内容`
### 各层级规则
范围：
1. 全局通用填common
2. 页面专属填页面英文缩写（如首页 = home、我的页面 = mine）。
分类：
1. 按功能 / 类型填固定关键词（如导航 = index、功能 = feature、按钮 = btn、单位 = unit、标题 = title）。
内容：
1. 用英文原词 / 简洁缩写（如运动 = workout、分 = min），确保无歧义。
### 命名示例
common_unit_kcal、
home_index_lifestyle、
home_btn_join。
### 禁止规则
1. 不超过三级层级；
2. 分类不自创，格式只用下划线；
3. 不使用模糊词汇（如 info、content）。

## Description 命名规则
### 核心原则
全中文表述、见名知义，与 Key 一一对应，无冗余。
### 命名格式
`范围/模块_分类_内容`
### 各部分规则
1. 范围/模块：全局通用填“通用”，页面专属填页面中文名（如首页、我的页面）。
2. 分类：与 Key 分类一致（如导航、功能、按钮、单位、标题）。
3. 内容：与 zh_CN 列中文一致，简洁明确。
### 示例
通用_单位_卡路里、首页_导航_生活、首页_按钮_加入。
### 禁止规则
1. 不超过三级表述；
2. 不用英文/缩写，不添加额外修饰；
3. 不使用模糊表述（如相关、内容）。

## 全局禁止规则
1. Key唯一性：除通用词汇，Key不重复
2. 禁止自创词汇：所有 en_US、zh_CN 内容必须来自截图英文或其准确翻译
3. 不得凭空创造：如截图无 “个人中心” 相关文本，不得新增 “home_module_profile”
4. 禁止修改原文：en_US 需完全匹配截图英文，包括拼写、格式（如截图为 “Al Tech”，不得改为 “AI Tech”）
5. 禁止格式混乱：严格遵循各字段格式要求，如 Key 的下划线分隔、Description 的 “业务分类_具体内容”、换行文本的 “\n” 保留
6. 特殊：截图中的“软键盘”等不要翻译
