### 0. 核心业务流程逻辑 (Business Logic Overview)
- **流程图结构**：[需求挖掘] → [用户分析] → [需求定义] → [原型验证] → [开发跟进] → [交付迭代]
- **节点详情**：
    1. **需求挖掘 (Discovery)**
        - *输入*：全网竞品评论、用户反馈数据。
        - *AI处理*：自动聚类分析痛点，量化需求优先级。
        - *输出*：机会点评估报告，避免“拍脑袋”决策。
    2. **用户分析 (Empathy)**
        - *输入*：目标用户画像标签。
        - *AI处理*：生成海量虚拟用户进行并发访谈，模拟全流程情绪曲线。
        - *输出*：深层需求洞察，消除主观偏见。
    3. **需求定义 (Definition)**
        - *输入*：需求草稿/框架。
        - *AI处理*：自动补全异常分支，生成业务流程图，结构化文档。
        - *输出*：逻辑自洽、无歧义的标准需求文档 (PRD)。
    4. **原型验证 (Prototyping)**
        - *输入*：自然语言描述 (Prompt)。
        - *AI处理*：生成可交互的网页 Demo 或高保真原型。
        - *输出*：零成本验证方案，快速试错，确认可行性。
    5. **开发跟进 (Delivery)**
        - *输入*：最终版 PRD + Demo。
        - *AI处理*：生成自动化测试脚本，扮演测试员穷举极端场景 (逻辑排雷)。
        - *输出*：高质量交付物，减少 Bug 返工。
    6. **交付迭代 (Iteration)**
        - *输入*：线上埋点数据。
        - *AI处理*：自然语言查询数据 (Text-to-SQL)，智能归因异常指标。
        - *输出*：数据驱动的迭代建议，形成闭环。

### 3. 产品工作流变革 (Comparison)
- **主题**：产品工作流变革
- **副标题**：从“单打独斗”到“人机协作”：AI 重塑产品全生命周期
- **布局**：横向闭环流程图 (Loop Flow)，强调迭代与反馈。
- **流程节点**：
    1. **需求挖掘 (Discovery)**：[舆情监控 → 竞品差异 → 机会评估]
        - *传统*：依赖滞后的研报，信息源单一；主观判断易陷入“伪需求”陷阱；缺乏量化数据支撑决策。
        - *AI*：AI 助手全网监控竞品/舆情；基于数据量化用户痛点；自动生成机会点评估报告。
    2. **用户分析 (Empathy)**：[深度访谈 → 问卷分析 → 画像构建]
        - *传统*：用户访谈样本少、周期长；定性分析受限于个人经验；难以覆盖目标用户场景。
        - *AI*：合成海量虚拟用户角色；批量并行访谈快速验证；消除主观偏见，挖掘深层需求。
    3. **需求定义 (Definition)**：[业务流程 → 异常分支 → 接口定义]
        - *传统*：需求文档纯文字晦涩难懂；特殊异常情况靠脑补；开发漏看细节导致返工。
        - *AI*：智能生成业务流程图；穷举异常流程规避风险；结构化文档确保逻辑自洽。
    4. **原型验证 (Prototyping)**：[低保真 → 高保真 → 交互动效]
        - *传统*：线框图交互性差，难体验；高保真依赖设计排期；验证想法的“试错成本”极高。
        - *AI*：一句话生成界面；可交互演示 Demo 即时体验；独立完成原型验证。
    5. **开发跟进 (Delivery)**：[技术评审 → 冒烟测试 → 视觉验收]
        - *传统*：需求传递损耗大，理解偏差；验收往往滞后于开发完成；Bug 修复挤占迭代时间。
        - *AI*：辅助生成测试计划与脚本；直接验收核心逻辑；前置质量卡点，减少返工。
    6. **交付迭代 (Iteration)**：[漏斗分析 → 留存归因 → A/B测试]
        - *传统*：埋点漏埋，取数排期长；归因分析靠猜，缺乏依据；难以形成有效的数据闭环。
        - *AI*：用大白话查数据；自动归因异常指标；基于数据生成迭代建议。

## 视觉与技术规范：

### 项目架构 (New Architecture)
- **入口**：`PPT模板.html` (负责加载资源与挂载 Root)
- **配置**：`js/data.js` (所有文本内容、图片路径配置)
- **组件**：`js/components.js` (图标、模态框等通用组件)
- **页面**：`js/slides.js` (各类型幻灯片的布局实现)
- **逻辑**：`js/app.js` (路由控制、键盘事件)
- **样式**：`css/style.css` (全局样式、动画定义)

### 样式风格 (CSS Variables)
- **背景**：`background: linear-gradient(135deg, #fdfbfb 0%, #ebedee 100%);` (模拟珍珠白质感)
- **主色**：`#32c48c` (AEKE Green - 用于强调文字、按钮、图表)
- **辅色**：`#2c3e50` (深灰 - 正文), `#ecf0f1` (浅灰 - 卡片背景)
- **字体**：系统无衬线字体 (San Francisco, Inter, Roboto, PingFang SC)，字重分明。
- **布局**：
    - 采用宽屏适配布局 (`max-w-[95vw]`)，优化投屏体验。
    - 预留 `.image-placeholder` 类，使用灰色虚线框占位，标注"Image Area"。
    - 卡片式设计 (Glassmorphism 玻璃拟态可选)。

### 交互细节
- 页面切换使用平滑的 `Fade + Scale` 组合动画。
- 所有横向滚动区域 (Timeline, Workflow) 均支持鼠标拖拽与滚轮映射。
- 列表项逐条浮现 (Fragmented lists)。
