# AI识别_电机纠错配置架构

## 硬件规格约束
- **电机配置**: 双电机 (左右独立)
- **力控范围**: 2.5kg - 50kg (单侧)
- **力控精度**: 0.5kg
- **数据频率**: 10Hz (100ms/帧)

## 设计原则
- **独立闭环**: 所有判定逻辑仅依赖电机传感器数据 (位置/速度/力)，绝不依赖视觉坐标，确保在无摄像头或视觉遮挡场景下仍能稳定运行。
- **冷启动支持**: 动态基准池具备“首次动作自动校准”能力，无需预设用户肢体参数即可开始纠错。
- **数据清洗**: 针对 10Hz 低频数据，底层强制进行微分平滑与噪声过滤，确保高阶指标 (加速度/力生成率) 的可用性。

## 全局逻辑控制
- **显示模式**：
  - 互斥（默认）：同一时刻仅显示一个最高优先级的纠错项。
  - 并行：同时显示所有触发的纠错项。
- **优先级排序**：
  - 升序（默认）：数值越小优先级越高 (P1 > P2)。
  - 降序：数值越大优先级越高。
- **优先级压制**：
  - 开启（默认）：当高优先级错误触发时，强制屏蔽低优先级错误的检测与显示。
  - 关闭：高优先级错误触发时，不影响低优先级错误的检测状态。
- **优先级挤占**：
  - 开启：允许新触发的错误挤占当前的纠错状态。
  - 关闭（默认）：一旦进入某种纠错状态，触发其他任何优先级的纠错都不挤占当前纠错，直到该纠错状态解除。
- **纠错交互触发频率**：
  - 配置数字（默认2）：决定几次错误触发纠错交互。

## 动作上下文输入 (新增)
- **说明**: 从动作库透传至算法层的静态元数据，用于适配不同动作的物理特性。
- **行程类型**:
  - *枚举*: `超短行程` | `短行程` | `中行程` (默认) | `长行程`
  - *定义*:
    - **超短行程**: 典型如提踵、耸肩 (总行程 < 15cm)。
    - **短行程**: 典型如二头弯举 (总行程 15-40cm)。
    - **中行程**: 典型如划船、推胸 (总行程 40-80cm)。
    - **长行程**: 典型如高位下拉、深蹲 (总行程 > 80cm)。
  - *作用*: 动态调整状态机的静止/抖动/触发阈值，解决小幅度动作无法触发的问题。

## 全局数据池
- **说明**：穷尽定义所有可用的物理量。底层引擎负责计算，上层配置直接引用 ID。
- **特性**：**数据即指标**。无需额外配置指标池。
- **命名规范**：
  - `[side]` 代表 `left` 或 `right`。
  - 引用时需完整写出，如 `left_force`。

### 1. 实时物理数- **说明**: 10Hz 实时输出的物理量，反映当前时刻的状态。
- **实时位置**:
  - ID: `[side]_pos`
  - 说明: 拉索拉出的绝对长度 (单位: cm)。
  - 精度: 0.1cm
  - 场景: 绝对位置判定 (如: 是否回位到起点、器械行程保护)。
- **实时行程**:
  - ID: `[side]_stroke`
  - 说明: 相对于动作起点的实时位移 (单位: cm)。
  - 公式: `当前位置 - 标准起点` (引用动态基准)
  - 场景: 判断动作幅度 (如: 下蹲深度、拉起高度)。
- **实时拉力**:
  - ID: `[side]_force`
  - 说明: 实时拉力/阻力 (单位: kg)。
  - 精度: 0.5kg
  - 场景: 实时负荷监控、离心阶段对抗力检测 (防止自由落体)。
- **实时速度**:
  - ID: `[side]_velocity`
  - 说明: 实时速度 (向心为正，离心为负，单位: m/s)。
  - 场景: 基于速度的训练核心指标、动作节奏控制 (快起慢放)。
- **实时加速度**:
  - ID: `[side]_acc`
  - 说明: 实时加速度 (单位: m/s²)。(10Hz下存在微分噪声，数值仅供参考)
  - 公式: `速度变化量 / 时间变化量`
  - 场景: 爆发力启动检测、动作停顿/粘滞点识别。
- **实时功率**:
  - ID: `[side]_power`
  - 说明: 实时输出的瞬时功率 (单位: W)。
  - 公式: `拉力(kg->N) * 速度(m/s)`
  - 场景: 爆发力训练核心指标 (如高翻、抓举)、最佳发力区间锁定。
- **实时力生成率**:
  - ID: `[side]_rfd`
  - 说明: 实时力生成率 (单位: kg/s)。(10Hz下存在微分噪声，数值仅供参考)
  - 公式: `拉力变化量 / 时间变化量`
  - 场景: 神经募集能力评估 (启动够不够快)、爆发力起步检测。

### 2. 过程统计数据
- **说明**: 基于当前相位(向心/离心)的实时统计值。进入新相位时自动重置，相位进行中实时更新。
- **峰值拉力**:
  - ID: `[side]_peak_force`
  - 说明: 相位内峰值拉力 (单位: kg)。
  - 场景: 评估绝对力量水平 (最大力量预估)、检测是否突破粘滞点。
- **峰值速度**:
  - ID: `[side]_peak_velocity`
  - 说明: 相位内峰值速度 (单位: m/s)。
  - 场景: 评估爆发力释放能力、判断是否利用惯性甩动。
- **峰值功率**:
  - ID: `[side]_peak_power`
  - 说明: 相位内峰值功率 (单位: W)。
  - 场景: 功率训练金标准、运动员表现评估。
- **平均拉力**:
  - ID: `[side]_avg_force`
  - 说明: 相位内平均拉力 (单位: kg)。
  - 公式: `累计冲量 / 时长`
  - 场景: 静力训练 (等长收缩) 质量评估、离心阶段持续对抗检测。
- **平均速度**:
  - ID: `[side]_avg_velocity`
  - 说明: 相位内平均速度 (单位: m/s)。
  - 公式: `行程 / 时长`
  - 场景: VBT 疲劳度监控 (速度下降 >20% 即视为力竭)、组间恢复评估。
- **平均功率**:
  - ID: `[side]_avg_power`
  - 说明: 相位内平均功率 (单位: W)。
  - 公式: `累计做功 / 时长`
  - 场景: 动作做功效率评估、高强度间歇训练 (HIIT) 强度监控。
- **做功**:
  - ID: `[side]_work`
  - 说明: 相位内累计做功 (单位: J)。
  - 公式: `拉力 * 位移 的积分`
  - 场景: 训练容量 (Volume) 精确计算、能量消耗估算。
- **冲量**:
  - ID: `[side]_impulse`
  - 说明: 相位内累计冲量 (单位: N·s)。
  - 公式: `拉力 * 时间 的积分`
  - 场景: 静力/离心训练负荷评估 (弥补位移为0时做功为0的缺陷)。

### 3. 动作特征数据
- **说明**: 描述动作结构、时序和质量的高级特征。
- **粘滞点速度**:
  - ID: `[side]_sticking_point_vel`
  - 说明: 粘滞点速度 (向心阶段速度最低点，单位: m/s)。
  - 场景: 动作瓶颈分析 (如卧推中途卡顿)、评估突破粘滞点的能力。
- **峰值力位置**:
  - ID: `[side]_peak_force_pos`
  - 说明: 峰值力发生的位置 (单位: cm)。
  - 场景: 发力模式分析 (如: 峰值过早可能意味着利用惯性甩起)。
- **力达峰时间**:
  - ID: `[side]_time_to_peak_force`
  - 说明: 从相位开始到达到峰值拉力的时长 (单位: s)。(受10Hz采样率限制，最小分辨率0.1s)
  - 场景: 爆发力构建速度评估 (力生成率的宏观体现，越短越好)。
- **推进相占比**:
  - ID: `[side]_propulsive_ratio`
  - 说明: 推进相占比 (加速度>0 的时间占比，单位: %)。(依赖加速度计算，受10Hz噪声影响)
  - 场景: 动作质量评估 (防止动作后半程“划水”或惯性投掷)。

### 4. 时间与节奏数据
- **说明**: 描述动作的时间结构。相位时长为实时值，单次时长为当前动作累积值。
- **向心时长**:
  - ID: `[side]_concentric_duration`
  - 说明: 向心阶段持续时间 (单位: s)。
  - 场景: 爆发力监测 (越短越好)、节奏控制 (如 1秒起)。
- **离心时长**:
  - ID: `[side]_eccentric_duration`
  - 说明: 离心阶段持续时间 (单位: s)。
  - 场景: 肌肥大训练核心 (如 3秒放)、离心控制力评估。
- **顶峰时长**:
  - ID: `[side]_peak_duration`
  - 说明: 顶峰阶段停留时间 (单位: s)。
  - 场景: 顶峰收缩质量检测 (如划船顶峰停顿)、等长收缩训练。
- **复位时长**:
  - ID: `[side]_reset_duration`
  - 说明: 复位/静止阶段停留时间 (单位: s)。
  - 场景: 动作间歇节奏控制、判断是否力竭休息过久。
- **单次时长**:
  - ID: `[side]_rep_duration`
  - 说明: 单次动作总耗时 (向心+离心+顶峰，单位: s)。
  - 场景: 动作总节奏监控 (如 3-0-1-0 节奏的总耗时)。
- **肌肉张力时长**:
  - ID: `[side]_tut`
  - 说明: 肌肉张力时长 (单位: s)。
  - 公式: `向心时长 + 离心时长 + 顶峰时长`
  - 场景: 肌肥大训练核心指标 (代谢压力累积评估)。
- **离向比**:
  - ID: `[side]_ecc_con_ratio`
  - 说明: 离心时长 / 向心时长 (单位: Ratio)。
  - 公式: `离心时长 / 向心时长`
  - 场景: 动作节奏执行度评估 (如是否做到了“慢放快起”)。

### 5. 双侧协同数据
- **说明**: 描述左右电机的配合情况。内置对齐策略。
- **总拉力**:
  - ID: `total_force`
  - 说明: 总拉力 (左+右，单位: kg)。
  - 公式: `左拉力 + 右拉力`
  - 策略: 实时求和。
  - 场景: 总负荷强度监测 (判断是否达到训练重量标准)。
- **总做功**:
  - ID: `total_work`
  - 说明: 总做功 (左+右，单位: J)。
  - 公式: `左做功 + 右做功`
  - 策略: 结果求和 (上一笔)。
  - 场景: 总训练容量统计、能量消耗计算。
- **力偏差率**:
  - ID: `force_balance_rate`
  - 说明: 左右力偏差率 (单位: %)。
  - 公式: `|左拉力 - 右拉力| / Max(左拉力, 右拉力, 2.5kg)` (分母下限为最小力控范围，防止低负载下的数值噪点)
  - 策略: 峰值对比 (Running Peak)。
  - 场景: 左右肌力平衡监测 (检测核心偏置或单侧代偿)。
- **行程偏差率**:
  - ID: `stroke_balance_rate`
  - 说明: 左右行程偏差率 (单位: %)。
  - 公式: `|左行程 - 右行程| / Max(左行程, 右行程, 1.0cm)` (分母下限1.0cm)
  - 策略: 峰值对比。
  - 场景: 动作幅度同步性监测 (检测肢体协调性)。
- **功率偏差率**:
  - ID: `power_balance_rate`
  - 说明: 左右功率偏差率 (单位: %)。
  - 公式: `|左功率 - 右功率| / Max(左功率, 右功率)`
  - 策略: 峰值对比。
  - 场景: 左右爆发力平衡监测 (高阶运动员关注指标)。

### 6. 系统状态数据
- **时间戳**:
  - ID: `timestamp`
  - 说明: 系统时间戳 (单位: ms)。
- **电机原始计次**:
  - ID: `[side]_raw_count`
  - 说明: 左右电机独立累计的物理计次 (用于调试或单侧训练分析)。
  - 逻辑: 每次满足 [单电机计次条件] 时 +1。
- **用户显示计次**:
  - ID: `user_display_count`
  - 说明: **最终呈现给用户的计次** (综合业务计次)。
  - 逻辑: 基于 [用户行为状态机] 和 [同步模式] 综合判定，过滤无效动作和预备动作。
- **当前计次**:
  - ID: `current_count`
  - 说明: 当前动作计次。
- **当前相位**:
  - ID: `current_phase`
  - 说明: 当前动作所处相位。
  - 枚举: 准备/向心/顶峰/离心/复位
- **相位时长**:
  - ID: `phase_duration`
  - 说明: 当前相位已持续时长 (单位: s)。
- **电机状态**:
  - ID: `motor_status`
  - 说明: 电机健康状态。
  - 枚举: 正常/过热/保护/故障
- **器械类型**:
  - ID: `accessory_type`
  - 说明: **物理约束输入**。仅作为设备连接状态参考，**不强制绑定计次逻辑**。无论连接何种器械，系统均基于左右电机的实时动态（时序/行程）自动判定单/双侧模式，以真实反映用户行为。
  - 枚举: 手柄 | 横杆
- **同步模式**:
  - ID: `sync_mode`
  - 说明: **评估策略选择器**。`双侧协同`关注同步性与平衡度；`单侧交替`关注独立性与峰值交错。直接决定纠错算法（如动力链型）的判定逻辑。
  - 枚举: 双侧协同 | 单侧交替 | 探测中

## 全局动态基准池
- **说明**：系统内置的自适应基准，由底层算法自动维护生命周期。上层配置直接引用 ID。
- **特性**：支持“初始化-锁定-修正”的动态闭环。

### 1. 位置与行程基准
- **标准起点**:
  - ID: `std_start_pos`
  - 逻辑: **多因子静止侦测**。
    - *条件1 (张力)*: `实时拉力 > 2.0kg` (确保绳索紧绷，非松弛态)。
    - *条件2 (位移稳定)*: 最近 1.0s (10帧) 窗口内，`Max(位置) - Min(位置) < 2.0cm` (利用位移极差抗 10Hz 噪点)。
    - *条件3 (低速确认)*: 最近 1.0s `|平均速度| < 0.05m/s` (辅助校验)。
  - **智能漂移修正**:
    - *允许更新*: 当前处于 [准备] 或 [复位] 状态，且满足静止侦测条件。此时将 `std_start_pos` 更新为当前位置（适配用户后退站位/调整握距）。
    - *禁止更新*: 当前处于 [起势(Setup)]、[向心]、[顶峰] 或 [离心] 状态。防止将动作中的停顿（如深蹲顶点、划船顶峰）误判为新起点。
  - 作用: 计算 `实时行程` 的基础。
- **标准行程**:
  - ID: `std_stroke`
  - 逻辑: **动态平均法**。
    - *冷启动*:
      - 第 1 次有效动作（满足起势判定后的首次完整动作）的行程直接作为临时标准行程。
      - 若第 1 次动作行程异常小（< 30% 预设参考值），则不更新，继续等待下一次。
    - *动态更新*:
      - 维护一个长度为 N (默认 5) 的历史行程队列。
      - 每次新动作完成，若行程在 `[0.5 * 当前标准行程, 1.5 * 当前标准行程]` 范围内，则入队。
      - `标准行程 = 队列中所有值的平均值`。
      - *最小行程保护*: 若计算出的 `标准行程` < `动作类型最小阈值` (如短行程动作 < 15cm)，强制使用预设最小值，防止因用户微动导致的基准坍塌。
    - *异常剔除*:
      - 若某次动作行程 < 50% 或 > 150% 当前标准行程，视为异常（半程或过拉），不计入历史队列，不影响标准行程更新。
  - 作用: 判断 `行程不足` 或 `幅度过大`。

### 2. 力量与爆发力基准
- **1RM估算值**:
  - ID: `est_1rm`
  - 逻辑: 基于当前负荷与速度估算的最大力量。
  - 作用: 推荐训练重量，判断强度是否达标。
- **历史1RM**:
  - ID: `history_1rm`
  - 逻辑: 记录用户历史最佳的 1RM 数据。
  - 作用: 长期力量进步监测，计算相对强度及今日状态评分。
- **历史峰值功率**:
  - ID: `history_peak_power`
  - 逻辑: 记录本组或历史最佳的峰值功率。
  - 作用: 爆发力训练的目标参考。

### 3. 疲劳与状态基准
- **组峰值速度**:
  - ID: `set_best_velocity`
  - 逻辑: 记录本组动作中出现的最高平均速度 (通常是第1-2次)。
  - 作用: 速度训练核心基准。当当前速度下降超过 20% (对比组峰值) 时，判定为力竭。
- **组峰值功率**:
  - ID: `set_best_power`
  - 逻辑: 记录本组动作中出现的最高平均功率。
  - 作用: 爆发力耐力监测。

### 4. 习惯与偏好基准
- **左右平衡常模**:
  - ID: `user_balance_norm`
  - 逻辑: 基于用户历史数据计算的左右力量/行程偏差率均值。
  - 作用: 区分“结构性偏科”与“突发性代偿”。若偏差率显著偏离常模，提示风险。
- **标准动作时长**:
  - ID: `std_rep_duration`
  - 逻辑: 基于历史 N 次有效动作的平均时长 (剔除异常值)。
  - 作用: 判断动作节奏是否忽快忽慢，或是否符合用户习惯。
- **标准发力曲线**:
  - ID: `std_force_curve`
  - 逻辑: 归一化后的历史平均发力波形。
  - 作用: `波形相似型` 纠错的核心基准，检测发力模式是否变形。

## 状态机配置体系

### 1. 单电机状态机 (底层物理层)
- **说明**: 针对左/右电机独立运行的状态流转，负责底层的物理相位识别。
- **阈值适配策略**:
  - **超短行程**: 静止区 `<1.5cm` | 抖动区 `1.5-2.5cm` | 触发阈值 `>2.5cm` | 回程阈值 `1.5cm`
  - **短行程**: 静止区 `<2.5cm` | 抖动区 `2.5-4.0cm` | 触发阈值 `>4.0cm` | 回程阈值 `2.5cm`
  - **中/长行程** (默认): 静止区 `<4.0cm` | 抖动区 `4.0-6.0cm` | 触发阈值 `>6.0cm` | 回程阈值 `4.0cm`
- **准备**:
  - *定义*: 动作未开始或处于预备态 (含静止与起势前的微动)。
  - *进入条件*:
    1. **系统初始化**: 默认状态。
    2. **动作循环结束**: 从 [复位] 状态自动流转进入。
    3. **未触发向心**: 实时行程始终在 `±触发阈值` 以内。
  - *区间定义*:
    - **静止区**: `0 - 静止阈值` (忽略微小漂移)
    - **预备/抖动区**: `静止阈值 - 触发阈值` (视为起势找感觉或调整站位，不触发动作开始，保持在准备态)。
- **向心**:
  - *定义*: 克服阻力，行程增加的阶段。
  - *进入条件*: `实时行程 > 触发阈值` AND (`速度 > 0.05m/s` OR `连续3帧行程增加`).
  - *锁定机制*: 一旦进入向心，除非满足[顶峰]或[离心]条件，否则即使速度跌至0（中途停顿/粘滞），仍保持向心状态。
  - *复合动作支持*: 若速度下降但 `行程` 仍在增加（或保持），视为动作延续，不切断状态。
- **顶峰**:
  - *定义*: 动作幅度的转折点区间。
  - *进入条件*:
    1. **自然转折**: `速度趋势由正转负` (需平滑处理) AND `实时行程 > 50%标准行程` (防止起始段抖动误判).
    2. **静态保持**: `|速度| < 0.05m/s` (近似静止) AND `实时行程 > 80%标准行程` (高位停顿).
  - *退出条件*:
    1. **转离心**: `实时行程` 较 [峰值位置] 回落超过 `回程阈值` (引用阈值适配策略).
    2. **假动作回向心**: `速度 > 0.05m/s` AND `行程 > 峰值位置`.
- **离心**:
  - *定义*: 顺应阻力，行程减小的阶段。
  - *进入条件*: `当前状态 == 顶峰` AND `实时行程` 较 [峰值位置] 回落超过 `回程阈值` (引用阈值适配策略)。
  - *抗噪策略*: 在 10Hz 低速离心时，允许速度在 0 附近波动，只要 `行程趋势` 总体向下，不打断离心状态。
  - *退出条件*:
    1. **动作触底**: 满足 [复位] 进入条件。
    2. **连续动作**: 在未复位的情况下，`速度` 再次转为正向 (> 0.05m/s) 且 `行程` 增加。-> 直接流转至 [向心]。
- **复位**:
  - *定义*: 动作结束回到起点。
  - *进入条件*: `实时行程 < 静止阈值` (回归原点) OR (`实时行程 < 30%标准行程` AND `|速度| < 0.05m/s` 保持 `> 0.5s`).
  - *动作*: 触发计次结算与数据归档，随后自动流转至 [准备] 状态。

### 2. 用户行为状态机 (顶层业务层)
- **说明**: 综合左右电机状态，输出用户当前的整体动作相位。用于驱动纠错逻辑的“检测时机”。
- **同步判定**:
  - **说明**: 仅用于在动作**起始阶段**识别用户意图（双侧协同 vs 单侧/交替），而非全过程的质量评估。
  - **判定依据**: 纯时序判定。计算 `向心启动时差` (`|左侧进入向心时刻 - 右侧进入向心时刻|`)。
  - *同步*: 时差 `≤ 0.2s` (探测期) 或 `≤ 0.35s` (锁定后)。
  - *异步*: 时差 `> 0.2s` (探测期) 或 `> 0.35s` (锁定后)。
- **起始姿态识别**:
  - **说明**: 自动识别动作是“低位起步”(如弯举) 还是 “高位起步”(如深蹲)，以解决第一下是否计次的问题。
  - **逻辑**:
    - **冷启动策略**: 若 `标准行程` 未建立 (第一笔动作)，**采用**绝对位移判定。
      - *参考值来源*: 依据 [动作上下文-行程类型] 获取预设典型值 (长行程=80cm, 中行程=50cm, 短行程=30cm)。
    - **低位预备**: 启动后，行程保持在 `< 20%` (冷启动时 `< 15cm`) 且稳定 > 1s。 -> 策略: **向心即开始**。
    - **高位预备**: 启动后，用户先拉起至 `> 60%` (冷启动时 `> 50%参考值`) 且稳定 > 1s (如深蹲起杠)。 -> 策略: **标记为起势(不计次)**，等待下一次完整的 [离心->向心] 循环。
- **状态流转**:
  - **准备**:
    - *定义*: 双侧均处于静止或调整状态。
    - *进入条件*: 双侧电机均处于 `准备` 或 `复位`。
  - **向心**:
    - *定义*: 任意一侧开始发力。
    - *进入条件*: 任意一侧电机进入 `向心`。
    - *模式识别*:
      - **双侧**: 双侧均 `向心` (且满足“同步”判定)。
      - **交替**: 一侧 `向心` 另一侧 `非静止(顶峰/离心)`，或双侧 `向心` 但满足“异步”判定。
      - **单侧**: 仅一侧 `向心` 另一侧 `准备/复位` (需延迟等待“同步”判定窗口结束)。
  - **顶峰**:
    - *定义*: 动作达到最大幅度或发力顶峰。
    - *进入条件*:
      - **双侧**: 任意一侧电机进入 `顶峰` (优先响应到达顶点的肢体，确保“向心->顶峰”状态流转闭环，容忍双侧微小时序偏差)。
      - **单侧/交替**: 活跃侧电机进入 `顶峰`。
  - **离心**:
    - *定义*: 动作还原阶段。
    - *进入条件*:
      - **双侧**: 任意一侧电机进入 `离心`。
      - **单侧/交替**: 活跃侧电机进入 `离心`。
  - **复位**:
    - *定义*: 动作结束，准备进入下一次循环。
    - *进入条件*:
      - **双侧**: 双侧电机均进入 `复位`。
      - **单侧/交替**: 活跃侧电机进入 `复位`。

### 3. 通用计次逻辑
- **说明**: 替代旧版计次逻辑，实现全场景闭环。
- **核心原则**: **计次发生在“顶峰”时刻 (心理预期峰值)**，且必须区分物理层与业务层。

#### A. 计次触发时机
- **触发事件**: 状态机从 `向心` 进入 `顶峰` (或直接转 `离心`) 的瞬间。
- **心理学依据**: 用户通常在发力结束、动作达到最大幅度时默念计数 ("1... 2...")，而非动作回到原点时。
- **防抖机制**: 触发后设置 `0.5s` 计次冷却时间，防止顶峰微小抖动导致连击。

#### B. 有效性判定
- **幅度达标**: `当前行程` ≥ `计次阈值` (计次阈值 = `0.8 * 标准行程`)。
  - *例外*: 若为 `超短行程`，绝对行程需 > 2.5cm。
- **起势过滤**:
  - 若当前处于 `高位预备` 模式，且这是第一次拉起 -> **判定为“起势”**，不计次，输出起势事件。
  - 若处于 `低位预备` 或 `正常循环` -> **判定为“有效次数”**，计次+1。
- **异常过滤**:
  - 若 `向心时长` < 0.2s (极速抽动) -> 忽略。
  - 若 `向心平均力` < 2kg (空拉) -> 忽略。

#### C. 综合计次策略
- **场景1: 双侧协同**
  - **逻辑**: 只有当 [用户行为状态机] 进入 `顶峰` 时触发。
  - **计数**: `用户显示计次 + 1`。
  - **说明**: 忽略左右电机微小的先后到达时间，统一为一个计数事件。
- **场景2: 单侧交替**
  - **逻辑**: 左右电机独立监听。
  - **计数**:
    - 左电机进顶峰 -> `用户显示计次 + 1`
    - 右电机进顶峰 -> `用户显示计次 + 1`
  - **说明**: 此时用户总计次 = 左+右。
- **场景3: 单侧独立**
  - **逻辑**: 仅监听活跃侧电机。
  - **计数**: 活跃侧进顶峰 -> `用户显示计次 + 1`。

#### D. 特殊场景闭环
- **力竭半程**:
  - **逻辑**: 若连续 2 次 `向心峰值` ∈ [50%, 80%] 且 `速度` 衰减。
  - **处理**: 计次 +1 (或 +0.5)，并标记为 *力竭半程* 状态。
- **中途调整**:
  - **逻辑**: 若动作中途停顿 > 3s (进入 `复位` 或 `准备`)。
  - **处理**: 重置 `起始姿态识别` 逻辑。下一次拉起重新判断是 起势 还是 正式动作。
- **起步确认**:
  - **逻辑**: 第一笔动作必须满足 `行程 > 50% 标准行程` (或绝对值 >15cm) 才会被视为 起势 或 正式动作，过滤用户试探性的微动。

## 有效性校验配置 (全局)
- **说明**: 定义动作计次的必要前置条件，过滤无效动作。
- **起点有效性**:
  - 逻辑: `|当前起点 - 基准起点| ≤ 10cm`
  - 作用: 防止用户大幅改变站位后，旧基准导致的误判。若不满足，不计次并触发基准修正检测。

## 错误项配置架构
- **说明**：定义单个错误项的完整配置结构，包含识别逻辑与交互反馈。

### 1. 基本信息
- **标识**：
  - *ID*：唯一标识符 (如 `range`)
  - *名称*：显示名称 (如 `行程不足`)
- **属性**：
  - *优先级*：P0 (阻断) | P1 (核心) | P2 (优化)
  - *算法分类*：标准指标型 | 动力链型 | 节奏速率型 | 波形相似型 | 一致性型 | 控制型

### 2. 核心算法配置 (多选一)

#### 类型A：标准指标型
- **适用场景**: 关注动作幅度是否达标、力量是否衰减。
- **属性定义**:
  - *目标数据*: 引用 [全局数据池] ID (如 `左侧行程`)。
  - *阈值*: 数值比较 (如 `> 80%`, `< 5cm`)。
  - *容忍度*: 定义不同难度等级下的判定阈值。
    - 格式：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *引导词* (非必填):
    - 格式：`条件 : 文案`
    - 动态阈值：使用 `{阈值}` 占位符代表当前生效的容忍度阈值。
    - 说明：支持多区间定义（含合格区间），系统根据实时指标值自动匹配触发。
- **配置示例**: 行程幅度
  - 目标数据: `左侧行程`
  - 阈值: `< 80%`
  - 容忍度: `严格 < 85% / 标准 < 80% / 宽松 < 75%`
  - 引导词:
    - `< {阈值} : "拉满行程，不要偷懒"`
    - `>= {阈值} : "幅度完美，控制节奏缓慢下放"`

#### 类型B：动力链型
- **适用场景**: 关注双手配合是否默契、发力是否同步。
- **属性定义**:
  - *目标数据*: 引用 [全局数据池] ID (如 `力偏差率`)。
  - *阈值*: 数值比较。
  - *容忍度*: `严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *引导词* (非必填):
    - 格式：`条件 : 文案`
    - 动态阈值：使用 `{阈值}` 占位符。
- **配置示例**: 左右同步
  - 目标数据: `力偏差率`
  - 阈值: `> 0.35s`
  - 容忍度: `严格 > 0.25s / 标准 > 0.35s / 宽松 > 0.45s`
  - 引导词:
    - `> {阈值} : "双手同时发力"`
    - `<= {阈值} : "节奏同步很好，保持住"`

#### 类型C：节奏速率型
- **适用场景**: 关注动作快慢节奏（如慢放快拉）。
- **属性定义**:
  - *目标数据*: 引用 [全局数据池] ID (如 `左侧离心时长`)。
  - *阈值*: 数值比较。
  - *容忍度*: `严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *引导词* (非必填):
    - 格式：`条件 : 文案`
- **配置示例**: 离心控制
  - 目标数据: `左侧离心时长`
  - 阈值: `< 1.0s`
  - 容忍度: `严格 < 1.5s / 标准 < 1.0s / 宽松 < 0.5s`
  - 引导词:
    - `< {阈值} : "慢一点下放，控制离心"`
    - `>= {阈值} : "节奏控制很好，继续保持"`

#### 类型D：波形相似型
- **适用场景**: 关注发力曲线的平滑度或与标准模板的相似性。
- **属性定义**:
  - *目标数据*: 引用 [全局数据池] ID (如 `左侧拉力` 波形)。
  - *参考波形*: `历史均值` | `标准模板`。
  - *阈值*: 相似度得分 (0.0 - 1.0)。
  - *容忍度*: `严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *引导词* (非必填):
    - 格式：`条件 : 文案`
- **配置示例**: 动作稳定性
  - 目标数据: `左侧拉力`
  - 阈值: `< 0.8`
  - 引导词:
    - `< {阈值} : "动作不要变形，保持轨迹一致"`

#### 类型E：一致性型 (新增)
- **适用场景**: 监测动作质量的衰减（疲劳管理）。
- **属性定义**:
  - *目标数据*: 引用统计数据 (如 `峰值速度`).
  - *参考窗口*: 最近 N 次 (如 3次).
  - *计算逻辑*: `(当前值 - 窗口均值) / 窗口均值`.
  - *阈值*: 下降百分比 (如 `-20%`).
- **配置示例**: 力竭预警
  - 目标数据: `左侧平均速度`
  - 阈值: `< -20%` (速度下降超过20%)
  - 引导词:
    - `< {阈值} : "速度慢了，加油顶住！"`

#### 类型F：控制型 (新增)
- **适用场景**: 离心阶段卸力、自由落体、器械回弹。
- **属性定义**:
  - *目标数据*: `实时拉力` 或 `离心失重率`.
  - *阈值*: 最小维持力 (公式: `Max(2.0kg, 0.1 * 设定阻力)`).
- **配置示例**: 离心卸力
  - 目标数据: `左侧拉力`
  - 阈值: `< 3.0kg` (设定阻力为20kg时，回放力小于3kg说明完全没抵抗)
  - 引导词:
    - `< {阈值} : "不要卸力，控制着放回去"`

### 3. 判定策略配置
- **检测时机**：
  - *枚举*：向心 | 离心 | 全程 | 启动瞬间 | 结束瞬间
  - *说明*：指定该错误项的激活时间窗口。
- **判定模式**：
  - *枚举*：一票否决 | 达成即过 | 实时跟随
- **时序防抖**：
  - *触发*：连续 N 次/帧触发。

### 4. 交互反馈配置
- **说明**：定义该错误触发时的视觉/听觉反馈。
- **设计原则**：采用 **“现状-差异-引导”** 的闭环设计。
- **状态：正确**：
  - *元素列表* (可配置多个)：
    - 类型：`提示词` | `特效` | `数值高亮`
    - 定义：内容 | 绑定数据
    - 样式：颜色（如绿色）、动效
    - **示例**：
      - `提示词`: 内容="幅度完美", 样式=绿色
      - `数值高亮`: 绑定=`左侧行程`, 样式=绿色脉冲
- **状态：错误**：
  - *元素列表* (可配置多个)：
    - 类型：`提示词` | `仪表盘` | `波形图` | `数值`
    - 定义：内容 | 绑定数据
    - 样式：颜色（如红色/橙色）、闪烁
    - **示例**：
      - `提示词`: 内容="拉满行程", 样式=红色
      - `仪表盘`: 绑定=`左侧速度`, 样式=红色刻度 (显示过快)
      - `波形图`: 绑定=`左侧拉力`, 样式=红色虚线 (显示掉力区域)
