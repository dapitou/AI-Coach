# 引导词编写规范

## 1. 核心定义
- **名称变更**：统称为 **“引导词”**，不再使用“提示词”。
- **定位**：动态的、过程导向的实时反馈。
- **角色**：像一位在旁陪练的专业教练，关注当下的状态和下一刻的趋势。
- **目的**：通过每个瞬间的实时引导，帮助用户正确、高质量地完成整个动作循环，而不是仅仅给出一个冷冰冰的“错误”判定。

## 2. 编写原则

### 2.1 全区间覆盖
必须覆盖指标的所有可能区间，形成正向反馈闭环：
- **错误区间**：给出具体的**纠正指令**（告诉用户怎么做）。
- **合格区间**：给出明确的**确认与下一步指引**（告诉用户做对了，并预告接下来的动作）。
- **无死区**：确保数值区间连续，不存在未定义的“沉默盲区”。

### 2.2 语气与风格
- **专业且有人情味**：避免机械的数值描述，使用“教练口吻”。
- **简洁有效**：短促有力，适合运动场景下的语音播报。
- **节奏匹配**：文案语速适配动作类型（爆发力动作短促，静态动作平稳）。
- **正向鼓励**：即使在纠错时，也要包含引导向上的意味；在正确时，给予及时的肯定。
- **地道中文**：符合中文口语习惯，自然流畅。

### 2.3 状态流转自然
不同状态之间的文案切换必须顺畅，具有承接关系，构建“对话感”：
- **从错误 -> 正确**：指令 -> 确认。
  - *错误态*：“继续向上抬肘，寻找发力感”（指令）
  - *正确态*：“高度完美，控制节奏缓慢下放”（确认+预告）
- **避免割裂感**：不要让两条文案听起来像两个互不相关的机器人。

## 3. 标准范例

### 场景：侧平举 - 抬肘高度
- **背景**：检测大臂与身体的夹角。标准区间为 80°~100°。
- **错误示范 (反面教材)**：
  - `< 80°` : "角度过低" (机械，无指导意义)
  - `[80°, 100°]` : (无提示) (缺失正向反馈)
  - `> 100°` : "角度过高" (机械，挫败感)
- **标准示范 (S级)**：
  - **`< 80°` (过低)** : `"继续向上抬肘，寻找发力感"` (指令：向上)
  - **`[80°, 100°]` (合格)** : `"高度完美，控制节奏缓慢下放"` (确认：完美 + 预告：下放)
  - **`> 100°` (过高)** : `"略微下放，与肩同高即可"` (指令：下放 + 标准：与肩同高)
