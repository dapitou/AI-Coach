# 计次体验提升分析报告 (V11.0.0)

> **说明**：本文档深度对比了 `电机计次逻辑_旧版` 与 `AI识别_电机纠错配置架构 V11.0.0`。V11.0.0 不仅仅是参数调整，而是底层物理引擎的**代际重构**。

## 1. 核心维度总览

| 维度 | 旧版 V1.0 | 新版 V11.0 | 核心差异解读 |
| :--- | :---: | :---: | :--- |
| **基准构建** | 相似度拟合 (滞后) | *物理信号侦测* | 旧版需凑齐2次相似动作才建立基准；新版基于力/速度实时锁定起点，即刻生效。 |
| **状态判定** | 阈值过线 (脆弱) | *迟滞状态机* | 旧版速度一掉就切状态；新版引入*迟滞比较器*与*锁定机制*，抵抗抖动与中途停顿。 |
| **首次体验** | 加载等待 (缺失) | *冷启动策略* | 旧版第一下显示“加载中”；新版基于*动作上下文*预设绝对值，第一下即精准计次。 |
| **计次时机** | 行程过线 (慢半拍) | *心理顶峰* | 旧版在动作回程或达标线触发；新版在*顶峰停顿*瞬间触发，打击感强。 |
| **协同逻辑** | 物理强绑定 (死板) | *行为动态解耦* | 旧版逻辑被器械模式锁死；新版完全基于*用户真实发力行为*判断单/双侧。 |
| **场景覆盖** | 常规 | 全场景 | 新版完美覆盖*起势、力竭半程、超短行程、超慢离心*等极限场景。 |

## 2. 深度优化点拆解

### 2.1 基准与初始化

#### ✅ 优化点1：告别“加载中”，第一下即计次
*   **旧版逻辑**：`动作初始化` = 邻近2次向心起点相似 + 邻近2次行程相似。
    *   *后果*：用户前两次动作被系统拿去“校准”了，界面显示“加载中”，经常出现做完第2下突然跳到“2”或“3”的跳变，用户无法确认第一下是否有效。
*   **新版解决**：*冷启动绝对值策略*。
    *   引入 `动作上下文` (如深蹲=长行程)，系统预知“大约要拉多长”。
    *   第一笔动作直接与预设的*绝对阈值* (如 >40cm) 比对。
    *   *体验*：推起即有数，零延迟反馈。

#### ✅ 优化点2：起点判定从“猜”变成“测”
*   **旧版逻辑**：依赖“两次起点位置相似”来反推“这里应该是起点”。若用户每次站位微调，系统会不断重置。
    *   *后果*：起点漂移，导致行程计算错误。
*   **新版解决**：*静止侦测法*。
    *   物理定义：`拉力 > 2.0kg` (紧绷) + **`1s内位移波动 < 2.0cm` (空间锁定)** = 确定的起点。
    *   *优势*：针对 10Hz 低频数据，摒弃了高噪点的瞬时速度判定，改用**窗口位移极差**，抗抖动能力提升 10 倍。
    *   *体验*：无需凑次数，拉紧停住（或站定）即自动锁定起点。

#### ✅ 优化点3：小幅度动作的“幽灵拒签”
*   **旧版痛点**：硬编码 `5cm` 静止阈值。对于提踵（总行程8cm），5cm 占了60%行程，极易被当做抖动过滤。
*   **新版解决**：*行程分级适配*。
    *   将动作分为 `超短/短/中/长` 四档。
    *   超短行程动作触发阈值降至 `> 2.5cm` (架构定义值)。
    *   *体验*：精准捕捉每一次耸肩和提踵，彻底解决小幅度动作漏计问题。

### 2.2 状态机与抗噪

#### ✅ 优化点4：状态机死锁，拒绝反复横跳
*   **旧版痛点**：依赖实时趋势。速度一掉（如动作中途卡顿），状态机就可能误判为结束或离心，将一个动作切成两半。
*   **新版解决**：*状态锁定*。
    *   一旦进入 `向心`，除非满足明确的 `顶峰` 或 `离心` 条件，否则*死锁*在向心态。
    *   *体验*：动作中途停顿、换气、粘滞点挣扎，都不会打断计次。

#### ✅ 优化点5：迟滞离心，稳如泰山
*   **旧版痛点**：速度 `< -0.1m/s` 即切离心。用户在顶点手抖一下（微小回落），就会触发“向心->离心->向心”的错误流转，导致多计次。
*   **新版解决**：*迟滞比较器*。
    *   要求回落超过明确的 *回程阈值* (如中长行程 > 4.0cm) 才确认进入离心。
    *   *体验*：顶峰无论怎么手抖，只要没真放下，状态就不乱。

### 2.3 业务逻辑与场景

#### ✅ 优化点6：双侧不同步的“宽容处理”
*   **旧版痛点**：双侧判定死抠 `0.2s` 时间差。若用户左右手稍有不同步（如时差0.3s），即会被*误判为单侧模式*，导致*多计1次*（1次双侧动作被拆分为左右各1次计次）。
*   **新版解决**：*前沿触发 + 模式锁定*。
    *   只要*任意一侧*进入“顶峰”，即视为动作达成（前沿触发）。
    *   一旦确认为双侧模式，容忍度放宽至 `0.35s`，兼容疲劳。
    *   *体验*：怎么拉都能算对，系统更懂人的疲劳。

#### ✅ 优化点7：起势的智能识别与相位对齐
*   **旧版痛点**：无法区分“深蹲起杠”和“一次有效动作”。导致深蹲“起杠”的那一下拉起被误计为第1次；且旧版无法自动适配“先离心后向心”（深蹲）与“先向心后离心”（弯举）的差异。
*   **新版解决**：*高位/低位起势判定 + 相位自适应*。
    *   **高位起势 (如深蹲)**：识别大幅度拉起并停顿，标记为 *起势* 状态。后续进入 *离心* (下蹲/回绳) 阶段，正式开启第1次动作循环（Setup -> 离心 -> 向心 -> 计次）。
    *   **低位起势 (如弯举)**：识别小幅度预紧，进入 *准备态*。等待行程突破触发阈值后，确认为 *向心* (拉起/出绳) 阶段，正式开启第1次动作循环（Setup -> 准备 -> 向心 -> 离心 -> 计次）。
    *   *体验*：系统自动听懂你是要“先蹲下去”还是要“先拉起来”。

#### ✅ 优化点8：力竭半程的补偿
*   **旧版痛点**：力竭时最后几次拉不到 80% 直接丢弃。用户明明拼尽全力拉了 70%，系统却毫无反应，挫败感极强。
*   **新版解决**：**力竭半程判定**。
    *   **逻辑**：识别到速度显著衰减 + 行程在 50%-80% 之间，判定为“力竭半程”。
    *   *体验*：给予计次（或标记为有效），认可用户的每一次努力。

#### ✅ 优化点9：器械模式解耦
*   **旧版痛点**：逻辑被“手柄/横杆”物理模式锁死。如果用长杆做单边划船，系统会因为检测不到另一侧而报错或不计次。
*   **新版解决**：*行为驱动逻辑*。
    *   `器械类型` 仅作参考。系统完全基于*左右电机的实际运动波形*来判断是单侧还是双侧。
    *   *体验*：用户可以自由发明训练方式（如单手拉横杆），算法都能适配。

## 3. 交互与反馈

#### ✅ 优化点10：心理学计次时机
*   **旧版**：在行程过线 (80%) 或 回程复位时 计次。
*   **新版**：在 *顶峰停顿* 瞬间计次。
*   **差异**：用户发力结束、动作达到最大幅度时，心理默念“1”。系统此刻同步发出“叮”的音效，打击感极强，**人机合一**。

#### ✅ 优化点11：多模态融合准备
*   **架构升级**：明确分离了 `用户显示计次` 与 `电机原始计次`。
*   **价值**：为未来接入视觉算法预留了接口。例如，视觉看到“人走了”而电机还在晃动，系统可以智能屏蔽无效计次，防止数据污染。

## 4. 总结
V11.0.0 版本的算法逻辑，不再是一个冷冰冰的“行程计数器”，而是一个能听懂肢体语言、包容疲劳变形、理解起势准备的**智能陪练**。它解决了“第一下不算”、“小动作不算”、“手抖算两次”、“力竭不算”等所有核心痛点。
