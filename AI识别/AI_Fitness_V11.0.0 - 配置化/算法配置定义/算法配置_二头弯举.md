# 算法配置：二头弯举 (Bicep Curl)

## 1. 基本信息
- **动作ID**：`bicep_curl`
- **动作名称**：二头弯举
- **版本**：V1.0
- **描述**：检测二头弯举过程中的弯举幅度和躯干借力情况。

## 2. 全局逻辑控制
- **优先级排序**：`升序` —— P1 > P2。
- **同时触发**：`互斥` —— 同时触发仅显示优先级最高的纠错交互。
- **优先级压制**：`开启` —— 当 P1 (幅度不足) 触发时，强制忽略 P2 (借力) 的错误显示。
- **优先级挤占**：`关闭` —— 纠错中，另一种纠错不可挤占。
- **纠错交互触发频率**：`2` —— 每2次错误触发纠错交互。

## 3. 虚拟计算点
我们需要定义一些骨骼模型中不存在的几何中心点，用于后续计算：
- **髋部中点**：
   - *用途*：计算躯干角度及借力晃动。
   - *算法*：`中点`
   - *输入*：左髋, 右髋
- **肩部中点**：
   - *用途*：计算躯干角度及借力晃动。
   - *算法*：`中点`
   - *输入*：左肩, 右肩

## 4. 动态基准
我们需要采集用户的身体数据作为参考基准：
- **最大臂长**：
   - *用途*：计算弯举幅度的压缩率基准。
   - *变量名*：`最大臂长`
   - *采集源*：
     - *算法*：`距离`
     - *输入*：肩关节, 腕关节
     - *参数*：模式=欧氏, 适配=双侧
   - *采集策略*：`智能基准校准`
     - 激活状态：`准备`
     - 阻尼系数：`0.05`
     - 衰减系数：`0.9995`
- **躯干长度**：
   - *用途*：计算躯干借力时的投影长度基准。
   - *变量名*：`躯干长度`
   - *采集源*：
     - *算法*：`距离`
     - *输入*：肩部中点, 髋部中点
     - *参数*：模式=欧氏
   - *采集策略*：`智能基准校准`
     - 激活状态：`准备`
     - 阻尼系数：`0.05`
     - 衰减系数：`0.9995`

## 5. 状态机配置
定义动作的生命周期流转：
- **准备**：
  - 说明：站立姿态，双臂自然下垂，动作就绪。
  - 逻辑：系统初始化或从复位状态判定稳定后进入。
  - 指标：`复位状态停留时长`
  - 阈值：`> 0.5s`
- **向心 (弯举)**：
  - 指标：`压缩率` (肩腕距离 / 最大臂长)
  - 阈值：`< 0.85` (开始向上弯举)
- **离心 (下放)**：
  - 指标：`压缩率`
  - 阈值：`> 0.90` (手臂回落)
- **复位**：
  - 说明：动作结束，回到初始站立位。
  - 指标：`压缩率`
  - 阈值：`> 0.95` (手臂完全伸直)
- **超时熔断**：
  - 目的：防止用户卡在弯举状态。
  - 条件：停留时长 `> 6.0秒` *且* 压缩率 `< 0.50`。
  - 动作：强制重置为 `准备` 状态。

## 6. 错误标准配置

### 错误项 A：抬腕幅度不足
- **ID**：`range`
- **优先级**：`P1`
- **算法类型**：`压缩率`
- **核心逻辑**：
  - **说明**：通过检测肩腕距离的缩短程度来判断弯举幅度（抗遮挡）。
  - 测量：`肩关节` 到 `腕关节` 的欧氏距离
  - 基准：`最大臂长` (准备状态下垂时采集)
  - 阈值：`< 0.35` (距离缩短至 35% 以内，即手腕非常接近肩膀)
  - 容忍度：`严格 0.30 / 标准 0.35 / 宽松 0.40`
  - 时序防抖：`3帧`
  - 单双侧：`触发：任一侧 / 解除：双侧达标`
- **判定模式**：`达成即过`
  - 逻辑：动作过程中只要有一次弯举至标准幅度，即判合格。

### 错误项 B：躯干借力 (晃动)
- **ID**：`swing`
- **优先级**：`P2`
- **算法类型**：`波动`
- **核心逻辑**：
  - **说明**：兼容正面与侧面视角。侧面检测躯干角度变化，正面检测躯干投影长度变化（利用透视原理）。
  - **逻辑关系**：`OR` (满足任一条件即触发)
  - **指标1 (侧面)**：`躯干垂直角` 变化幅度 > 10°
  - **指标2 (正面)**：`躯干投影长度` (肩髋距离) 变化幅度 > 10% (相对于动作底部的长度)
  - **阈值**：`波动 > 0.10` (角度或长度波动超过 10)
  - 容忍度：`严格 0.08 / 标准 0.10 / 宽松 0.15`
  - 时序防抖：`5帧`
  - 单双侧：`整体`
- **判定模式**：`一票否决`
  - 逻辑：全程只要出现大幅晃动（满足防抖），即判不合格。

## 7. 纠错交互配置

### 引导交互 A：抬腕幅度不足
- **绑定条件**：`range`
- **状态：错误**
  - 左引导：
    - 类型：`箭头`
    - 定义：起点 `左腕` -> 终点 `左肩`
    - 方向：`指向目标`
    - 样式：红色
    - 动效：`弹性`
  - 右引导：
    - 类型：`箭头`
    - 定义：起点 `右腕` -> 终点 `右肩`
    - 方向：`指向目标`
    - 样式：红色
    - 动效：`弹性`
  - 纠错提示：
    - 内容：手腕尽量靠近肩膀，顶峰收缩！
    - 样式：红色
- **状态：正确**
  - 左腕：
    - 类型：`对勾`
    - 位置：`左腕`
    - 样式：绿色
  - 右腕：
    - 类型：`对勾`
    - 位置：`右腕`
    - 样式：绿色
  - 提示词：
    - 内容：完美动作！
    - 样式：绿色

### 引导交互 B：躯干借力 (晃动)
- **绑定条件**：`swing`
- **状态：错误**
  - 髋部固定点：
    - 类型：`圆点`
    - 位置：`髋中点`
    - 样式：红色、实心
    - 动效：`呼吸`
  - 躯干线：
    - 类型：`实线`
    - 定义：`髋中点` -> `肩中点`
    - 样式：红色
  - 目标线：
    - 类型：`目标线`
    - 定义：`髋中点` -> `垂直向上`
    - 样式：绿色、虚线
  - 纠错提示：
    - 内容：核心收紧，身体不要晃动！
    - 样式：红色
