# 算法配置：弓步 (Lunge)

## 1. 基本信息
- **动作ID**：`lunge`
- **动作名称**：弓步
- **版本**：V1.0
- **描述**：检测弓步蹲过程中的膝关节稳定性、下蹲幅度和躯干中立位。

## 2. 全局逻辑控制
- **优先级排序**：`升序` —— P1 > P2 > P3。
- **同时触发**：`互斥` —— 同时触发仅显示优先级最高的纠错交互。
- **优先级压制**：`开启` —— 当 P1 (膝内扣) 触发时，强制忽略 P2/P3 的错误显示。
- **优先级挤占**：`关闭` —— 纠错中，另一种纠错不可挤占。
- **纠错交互触发频率**：`2` —— 每2次错误触发纠错交互。

## 3. 虚拟计算点
我们需要定义一些骨骼模型中不存在的几何中心点，用于后续计算：
- **髋部中点**：
   - *用途*：计算下蹲深度、躯干侧倾及动态基准。
   - *算法*：`中点`
   - *输入*：左髋, 右髋
- **肩部中点**：
   - *用途*：计算躯干侧倾（相对于髋部）。
   - *算法*：`中点`
   - *输入*：左肩, 右肩
- **膝盖中点**：
   - *用途*：计算站立基准高度。
   - *算法*：`中点`
   - *输入*：左膝, 右膝
- **前脚底中心**：
   - *用途*：绘制膝内扣时的垂直参考线起点。
   - *算法*：`中点` (需结合状态机判定前脚，此处定义通用逻辑)
   - *输入*：左踝, 左脚尖 (或 右踝, 右脚尖)
- **前膝理想投影**：
   - *用途*：绘制膝内扣时的垂直参考线终点。
   - *算法*：`投影`
   - *输入*：源=前膝, 参考=前脚底中心
   - *参数*：方向=垂直

## 4. 动态基准
我们需要采集用户的身体数据作为参考基准：
- **站立基准高度**：
   - *用途*：计算下蹲深度压缩率。
   - *变量名*：`站立基准高度`
   - *采集源*：
     - *算法*：`距离`
     - *输入*：髋部中点, 膝盖中点
     - *参数*：模式=垂直
   - *采集策略*：`智能基准校准`
     - 激活状态：`准备`
     - 阻尼系数：`0.01`
     - 衰减系数：`0.9995`

## 5. 状态机配置
定义动作的生命周期流转：
- **准备**：
  - 说明：双脚并拢站立，动作就绪。
  - 逻辑：系统初始化或从复位状态判定稳定后进入。
  - 指标：`复位状态停留时长`
  - 阈值：`> 0.5s`
- **向心 (下蹲)**：
  - 指标：`压缩率` (髋膝垂直距离 / 基准)
  - 阈值：`< 0.85`
- **离心 (起立)**：
  - 指标：`压缩率`
  - 阈值：`> 0.90`
- **进入纠错区间**：
  - 指标：`压缩率`
  - 阈值：`< 0.35` (进入弓步底部，大腿接近水平)
- **离开纠错区间**：
  - 指标：`压缩率`
  - 阈值：`> 0.35`
- **复位**：
  - 说明：动作结束回到站立。
  - 指标：`压缩率`
  - 阈值：`> 0.92`
- **超时熔断**：
  - 目的：防止用户卡在下蹲状态。
  - 条件：停留时长 `> 6.0秒` *且* 压缩率 `> 0.70`。
  - 动作：强制重置为 `准备` 状态。

## 6. 错误标准配置

### 错误项 A：膝内扣（弓步）
- **ID**：`valgus`
- **优先级**：`P1`
- **算法类型**：`比例` (或 偏移量)
- **核心逻辑**：
  - **说明**：弓步时前膝不应内塌，需保持与脚尖同向。
  - 分子：`双膝间距` (或 前膝X偏离)
  - 分母：`双髋间距`
  - 阈值：`最小 1.0` (弓步站距较窄，膝宽不应显著小于髋宽)
  - 容忍度：`严格 1.1 / 标准 1.0 / 宽松 0.9`
  - 时序防抖：`5帧`
  - 单双侧：`整体` (算法自动锁定前腿)
- **判定模式**：`过程修正`
  - 逻辑：进入纠错区间后，必须修正动作并保持正确，方可判合格。
- **纠错区间**：
  - 约束指标：`压缩率`
  - 约束阈值：`< 0.35`

### 错误项 B：下蹲幅度不足（弓步）
- **ID**：`depth`
- **优先级**：`P2`
- **算法类型**：`压缩率`
- **核心逻辑**：
  - **说明**：前腿大腿应蹲至平行地面。
  - 测量：`髋部中点` 到 `前膝` (近似膝中点) 的垂直距离
  - 基准：`站立基准高度`
  - 阈值：`最大 0.20` (距离缩短至站立时的 20% 以内，即髋膝接近水平)
  - 容忍度：`严格 0.15 / 标准 0.20 / 宽松 0.25`
  - 时序防抖：`3帧`
  - 单双侧：`整体`
- **判定模式**：`达成即过`
  - 逻辑：只要在动作过程中有几帧达到了标准，本次动作即判合格。

### 错误项 C：躯干重心侧倾（通用）
- **ID**：`lean`
- **优先级**：`P3`
- **算法类型**：`偏移率`
- **核心逻辑**：
  - **说明**：检测躯干是否向左或向右倾斜。
  - 测量：`肩部中点` X坐标 相对于 `髋部中点` X坐标 的偏移量
  - 基准：`躯干长度` (肩中点到髋中点的直线距离)
  - 阈值：`> 0.10` (偏移量超过躯干长度的 10%)
  - 容忍度：`严格 0.08 / 标准 0.10 / 宽松 0.15`
  - 时序防抖：`5帧`
  - 单双侧：`整体`
- **判定模式**：`一票否决`
  - 逻辑：全程只要出现侧倾（满足防抖），即判不合格。

## 7. 纠错交互配置

### 引导交互 A：膝内扣（弓步）
- **绑定条件**：`valgus`
- **状态：错误**
  - 前膝：
    - 类型：`箭头`
    - 定义：起点 `前膝`，方向 `向外`
    - 样式：红色
    - 动效：`弹性`
  - 参考线：
    - 类型：`参考线`
    - 定义：`前脚底中心` -> `前膝理想投影`，方向：`垂直`
    - 样式：绿色、虚线
  - 纠错提示：
    - 内容：膝与脚尖保持同向！
    - 样式：红色
- **状态：正确**
  - 前膝：
    - 类型：`对勾`
    - 位置：`前膝`
    - 样式：绿色

### 引导交互 B：下蹲幅度不足（弓步）
- **绑定条件**：`depth`
- **状态：错误**
  - 引导：
    - 类型：`箭头`
    - 定义：起点 `髋中点` -> 终点 `前膝` (垂直向下)
    - 样式：红色
    - 动效：`弹性`
  - 纠错提示：
    - 内容：蹲至前腿平行地面效果更好！
    - 样式：红色
- **状态：正确**
  - 髋部：
    - 类型：`对勾`
    - 位置：`髋中点`
    - 样式：绿色
  - 提示词：
    - 内容：完美动作！
    - 样式：绿色

### 引导交互 C：躯干重心侧倾（通用）
- **绑定条件**：`lean`
- **状态：错误**
  - 躯干线：
    - 类型：`实线`
    - 定义：`髋中点` -> `肩中点`
    - 样式：红色
  - 目标线：
    - 类型：`目标线`
    - 定义：`髋中点` -> `垂直向上`
    - 样式：绿色、虚线
  - 引导：
    - 类型：`箭头`
    - 定义：起点 `肩中点` -> 指向 `目标线`
    - 样式：红色
    - 动效：`弹性`
  - 纠错提示：
    - 内容：请保持身体不要侧倾！
    - 样式：红色
