# 算法配置：侧平举 (Lateral Raise)

## 1. 基本信息
- **动作ID**：`lateral_raise`
- **动作名称**：侧平举
- **版本**：V1.0
- **描述**：检测侧平举过程中的抬肘高度、耸肩代偿及肘部锁定情况。

## 2. 全局逻辑控制
- **优先级排序**：`升序` —— P1 > P2 > P3。
- **同时触发**：`互斥` —— 同时触发仅显示优先级最高的纠错交互。
- **优先级压制**：`开启` —— 当 P1 (抬肘高度) 触发时，强制忽略 P2/P3 的错误显示。
- **优先级挤占**：`关闭` —— 纠错中，另一种纠错不可挤占（无论优先级），直到上一个纠错解除。
- **纠错交互触发频率**：`2` —— 每2次错误触发纠错交互。

## 3. 虚拟计算点
我们需要定义一些骨骼模型中不存在的几何中心点，用于后续计算：
- **肩部中点**：
   - *用途*：计算耸肩幅度和动态基准高度。
   - *算法*：`中点`
   - *输入*：左肩, 右肩
- **左肩水平延伸点**：
   - *用途*：绘制抬肘高度不足时的目标参考点。
   - *算法*：`延伸`
   - *输入*：起点=左肩
   - *参数*：方向=X轴负向, 长度=[左肩, 左肘]
- **右肩水平延伸点**：
   - *用途*：绘制抬肘高度不足时的目标参考点。
   - *算法*：`延伸`
   - *输入*：起点=右肩
   - *参数*：方向=X轴正向, 长度=[右肩, 右肘]

## 4. 动态基准
我们需要采集用户的身体数据作为参考基准：
- **肩部基准高度**：
   - *用途*：计算耸肩幅度的压缩率基准。
   - *变量名*：`肩部基准高度`
   - *采集源*：
     - *算法*：`距离`
     - *输入*：肩部中点, 鼻子
     - *参数*：模式=垂直
   - *采集策略*：`智能基准校准`
     - 激活状态：`准备`
     - 阻尼系数：`0.05`
     - 衰减系数：`0.9995`

## 5. 状态机配置
定义动作的生命周期流转：
- **准备**：
  - 说明：双臂自然下垂，动作就绪状态。
  - 逻辑：系统初始化或从复位状态判定稳定后进入。
  - 指标：`复位状态停留时长`
  - 阈值：`> 0.5s`
- **向心 (举起)**：
  - 指标：`大臂垂直角` (任一侧)
  - 阈值：`> 30°`
 - **离心 (下放)**：
   - 指标：`大臂垂直角` (双侧)
   - 阈值：`< 30°`
 - **进入纠错区间**：
   - 指标：`大臂垂直角`
   - 阈值：`> 75°` (进入侧平举顶峰，大臂接近水平)
 - **离开纠错区间**：
   - 指标：`大臂垂直角`
   - 阈值：`< 70°` (离开侧平举顶峰)
- **复位**：
  - 指标：`大臂垂直角` (双侧)
  - 阈值：`< 20°`
- **超时熔断**：
  - 目的：防止用户卡在举起状态。
  - 条件：停留时长 `> 5.0秒` *且* 大臂垂直角 `< 30°`。
  - 动作：强制重置为 `准备` 状态。

## 6. 错误标准配置

### 错误项 A：抬肘幅度不足
- **ID**：`range`
- **优先级**：`P1`
- **算法类型**：`角度`
- **核心逻辑**：
  - 测量：大臂(肩-肘)与垂直线的夹角
  - 阈值：`> 80°` (接近水平)
  - 容忍度：`严格 85° / 标准 80° / 宽松 70°`
  - 时序防抖：`3帧`
  - 单双侧：`触发：任一侧不达标 / 解除：双侧均达标`
- **判定模式**：`达成即过`
  - 逻辑：动作过程中只要有一次抬高到标准角度，即判合格。

### 错误项 B：双肘展开幅度不足
- **ID**：`forward_drift`
- **优先级**：`P2`
- **算法类型**：`比例`
- **核心逻辑**：
  - **说明**：利用双肘间距与肩宽的比例检测动作宽度。侧平举时双肘应远离身体，若向前漂移，投影间距会显著缩小。
  - 分子：`双肘间距` (左肘到右肘的欧氏距离)
  - 分母：`双肩间距` (左肩到右肩的欧氏距离)
  - 阈值：`< 2.0` (标准侧平举通常 > 2.5，前平举约 1.0)
  - 容忍度：`严格 2.2 / 标准 2.0 / 宽松 1.8`
  - 时序防抖：`5帧`
  - 单双侧：`整体`
- **判定模式**：`一票否决`
  - 逻辑：全程只要出现明显的前平举趋势，即判不合格。

### 错误项 C：耸肩（通用）
- **ID**：`shrug`
- **优先级**：`P3`
- **算法类型**：`压缩率`
- **核心逻辑**：
  - 测量：`肩部中点` 到 `鼻子` 的垂直距离
  - 基准：`肩部基准高度`
  - 阈值：`最小 0.8` (距离缩短至 80% 以内，即压缩 > 20%)
  - 容忍度：`严格 0.85 / 标准 0.8 / 宽松 0.75`
  - 时序防抖：`5帧`
  - 单双侧：`整体` (基于肩部中点计算)
- **判定模式**：`一票否决`
  - 逻辑：全程只要出现耸肩，即判不合格。

### 错误项 D：肘部锁定
- **ID**：`elbow_lock`
- **优先级**：`P2`
- **算法分类**：动力链型-同步型
- **核心算法配置**：
  - 基准指标：`肩外展角`
  - 跟随指标：`肘屈曲角`
  - 拟合公式：`| 肘屈曲角 - (肩外展角 * 0 + 150) | ≤ 容忍阈值`
  - 容忍阈值：`严格 ≤ 5° / 标准 ≤ 10° / 宽松 ≤ 15°`
- **判定策略配置**：
  - 检测时机：全程
  - 单双侧逻辑：分侧
    - 触发逻辑：任一侧不达标
    - 解除逻辑：双侧均达标
  - 时序防抖：3帧
- **判定模式**：`一票否决`
- **交互反馈配置**：
  - 状态：错误
    - 元素名称：
      - 类型：`提示词`
      - 内容：保持肘部微屈固定，不要伸缩！
      - 样式：红色
