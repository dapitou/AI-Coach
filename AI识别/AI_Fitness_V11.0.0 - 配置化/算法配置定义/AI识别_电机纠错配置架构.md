# AI识别_电机纠错配置架构

## 全局逻辑控制
- **显示模式**：
  - 互斥（默认）：同一时刻仅显示一个最高优先级的纠错项。
  - 并行：同时显示所有触发的纠错项。
- **优先级排序**：
  - 升序（默认）：数值越小优先级越高 (P1 > P2)。
  - 降序：数值越大优先级越高。
- **优先级压制**：
  - 开启（默认）：当高优先级错误触发时，强制屏蔽低优先级错误的检测与显示。
  - 关闭：高优先级错误触发时，不影响低优先级错误的检测状态。
- **优先级挤占**：
  - 开启：允许新触发的错误挤占当前的纠错状态。
  - 关闭（默认）：一旦进入某种纠错状态，触发其他任何优先级的纠错都不挤占当前纠错，直到该纠错状态解除。
- **纠错交互触发频率**：
  - 配置数字（默认2）：决定几次错误触发纠错交互。

## 核心算子枚举 (电机专用)
- **说明**：处理一维时序信号（位置/力/速度）的原子算法。

### 1. 数值计算类
- **差值**:
  - 逻辑: `A - B`
  - 示例: `左右同步差 = 差值(左启动时间, 右启动时间)`
- **绝对值**:
  - 逻辑: `|A|`
  - 示例: `起点偏差绝对值 = 绝对值(当前起点 - 初始起点)`
- **极值**:
  - 逻辑: 窗口内或行程内的最大/最小值。
  - 示例: `向心峰值力 = 极值(力信号, 模式=最大, 窗口=向心区间)`
- **均值**:
  - 逻辑: 窗口内或多次动作的平均值。
  - 示例: `基准行程 = 均值(历史前3次有效行程)`

### 2. 逻辑判定类
- **阈值判定**:
  - 逻辑: `数值 > 或 < 设定值`
  - 特性: 支持绝对值(cm)与相对值(%)双模式。
- **趋势分析**:
  - 逻辑: 判定向心/离心/静止状态。
  - 参数: 抖动阈值(3-5cm), 有效阈值(>5cm)。
- **相似度**:
  - 逻辑: 比较当前行程波形与基准波形的重合度或数值接近度。
  - 示例: `行程相似度 = |A - B| / A`

## 信号源定义
- **物理信号**:
  - `左电机位置` / `右电机位置`: 电机拉线长度 (cm)
  - `左电机拉力` / `右电机拉力`: 实时拉力 (kg)
  - `左电机速度` / `右电机速度`: 实时速度 (m/s)
- **系统信号**:
  - `时间戳`: 时间戳 (ms)
  - `当前计次`: 当前计次
  - `当前相位`: 当前相位 (准备/向心/顶峰/离心/复位)

## 指标池配置
- **说明**：基于信号源计算的二阶指标，供下游模块复用。
- **基本属性**：
  - *ID*：指标唯一标识 (如 `向心行程`, `启动时间差`)
  - *名称*：显示名称
  - *算法*：引用 [核心算子枚举]
  - *输入*：依赖的信号源或其他指标
  - *参数* (非必填)：算法参数
- **配置示例**：
  - *ID*: `向心行程`
  - *算法*: `差值` (向心终点位置 - 向心起点位置)
  - *ID*: `起点偏差`
  - *算法*: `差值` (当前向心起点 - 基准起点)

## 动态基准池配置 (自适应核心)
- **说明**：电机逻辑的核心，支持“初始化-锁定-修正”的动态闭环。
- **基本属性**：
  - *ID*：基准唯一标识 (如 `标准行程`)
  - *来源指标*：引用 [指标配置] ID
- **采集策略**：
  - **初始化**:
    - 条件: `邻近N次偏差 ≤ 阈值` (如 N=2, 偏差≤5cm)
    - 动作: 锁定均值为基准。
  - **自适应修正**:
    - 条件: `连续N次偏差 > 触发阈值 AND 偏差稳定性 ≤ 稳定阈值`
    - 动作: 更新基准为最新均值。
    - 示例: 站位调整导致起点漂移，系统自动更新起点基准。

## 状态机配置
- **准备**:
  - 说明: 动作未开始或处于预备态。
  - 逻辑: 位移变化在 `±3cm` 以内，且持续时间满足阈值。
- **向心**:
  - 说明: 发力拉动阶段。
  - 触发: 位移变化 `> 5cm` (有效拉动)。
  - 属性: 记录启动时间戳、起点位置。
- **顶峰**:
  - 说明: 动作顶点停顿状态 (顶峰收缩)。
  - 触发: 向心结束，速度趋近于 0 (或进入死区)，且位置处于行程高点。
  - 作用: 用于检测“顶峰收缩时长”或“静态保持”。
- **离心**:
  - 说明: 放回阶段。
  - 触发: 速度反向 (负向速度超过阈值)，位移明显回落。
- **复位**:
  - 说明: 动作结束回到起点。
  - 触发: 离心结束，位移回到起点附近 (如 `< 20% 行程`)。
  - 动作: 触发计次判定逻辑 (`向心过线` AND `离心过线`)。

## 单双侧判定逻辑
- **说明**：定义当前动作是单侧交替还是双侧同时，以及错误判定的合并逻辑。
- **模式识别**:
  - **探测期**: 左右启动时间差 `≤ 0.2s` (双侧) | `> 0.2s` (单侧)。
  - **锁定后**: 双侧模式下容差放宽至 `0.35s` (适应疲劳)。
- **错误触发逻辑** (双侧模式下):
  - *任一侧不达标*：只要有一侧数据异常即报错 (如左右手任一侧行程不足)。
  - *双侧均不达标*：两侧同时异常才报错。

## 错误判定配置
- **说明**：定义动作评价的时间维度判定逻辑。
- **枚举**：
  - **一票否决**: 只要检测到瞬间超标即判错 (如: 启动不同步)。
  - **达成即过**: 在特定区间内达到标准即判对 (如: 行程拉满)。
  - **实时跟随**: 状态随数据实时跳变 (如: 速度仪表盘)。

## 错误项配置架构
- **说明**：定义单个错误项的完整配置结构，包含识别逻辑与交互反馈。

### 1. 基本信息
- **标识**：
  - *ID*：唯一标识符 (如 `range`)
  - *名称*：显示名称 (如 `行程不足`)
- **属性**：
  - *优先级*：P0 (阻断) | P1 (核心) | P2 (优化)
  - *算法分类*：标准指标型 | 动力链型 | 节奏速率型

### 2. 核心算法配置 (多选一)

#### 类型A：标准指标型
- **适用场景**: 关注动作幅度是否达标、力量是否衰减。
- **属性定义**:
  - *目标指标*: `向心行程`
  - *基准* (可选): `标准行程`
  - *阈值*: `< 80%` (或绝对值差 > N cm)
  - *容忍度*: `严格 / 标准 / 宽松`
  - *引导词*:
    - 格式：`条件 : 文案`
    - 要求：全区间覆盖，无死区；合格区间给出正向确认与下一步指引。
- **配置示例**: 行程幅度
  - 目标: `向心行程`
  - 阈值: `< 80%`
  - 引导词:
    - `< 80% : "拉满行程，不要偷懒"`
    - `>= 80% : "幅度完美，控制节奏缓慢下放"`

#### 类型B：动力链型
- **适用场景**: 关注双手配合是否默契、发力是否同步。
- **属性定义**:
  - *目标指标*: `启动时间差`
  - *阈值*: `> 0.35s` (基于锁定后的宽容阈值)
  - *引导词*:
    - `> 0.35s : "双手同时发力"`
    - `<= 0.35s : "节奏同步很好，保持住"`

#### 类型C：节奏速率型
- **适用场景**: 关注动作快慢节奏（如慢放快拉）。
- **属性定义**:
  - *目标指标*: `离心时长`
  - *阈值*: `< 1.0s`
  - *引导词*:
    - `< 1.0s : "慢一点下放，控制离心"`
    - `>= 1.0s : "节奏控制很好，继续保持"`

### 3. 判定策略配置
- **检测时机**：
  - *枚举*：向心 | 离心 | 全程 | 启动瞬间 | 结束瞬间
  - *说明*：指定该错误项的激活时间窗口。
- **判定模式**：
  - *枚举*：一票否决 | 达成即过 | 实时跟随
- **时序防抖**：
  - *触发*：连续 N 次/帧触发。

### 4. 交互反馈配置
- **状态：错误**：
  - *元素列表*：
    - 类型：`提示词` | `仪表盘变色` | `波形高亮`
    - 样式：颜色（如红色）、闪烁
- **状态：正确**：
  - *元素列表*：
    - 类型：`提示词` | `特效反馈`
    - 样式：颜色（如绿色）
