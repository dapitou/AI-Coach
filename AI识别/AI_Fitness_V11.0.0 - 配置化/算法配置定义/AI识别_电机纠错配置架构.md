# AI识别_电机纠错配置架构

## 硬件规格约束
- **电机配置**: 双电机 (左右独立)
- **力控范围**: 2.5kg - 50kg (单侧)
- **力控精度**: 0.5kg
- **数据频率**: 10Hz (100ms/帧)

## 全局逻辑控制
- **显示模式**：
  - 互斥（默认）：同一时刻仅显示一个最高优先级的纠错项。
  - 并行：同时显示所有触发的纠错项。
- **优先级排序**：
  - 升序（默认）：数值越小优先级越高 (P1 > P2)。
  - 降序：数值越大优先级越高。
- **优先级压制**：
  - 开启（默认）：当高优先级错误触发时，强制屏蔽低优先级错误的检测与显示。
  - 关闭：高优先级错误触发时，不影响低优先级错误的检测状态。
- **优先级挤占**：
  - 开启：允许新触发的错误挤占当前的纠错状态。
  - 关闭（默认）：一旦进入某种纠错状态，触发其他任何优先级的纠错都不挤占当前纠错，直到该纠错状态解除。
- **纠错交互触发频率**：
  - 配置数字（默认2）：决定几次错误触发纠错交互。

## 核心算子枚举 (电机专用)
- **说明**：处理一维时序数据（位置/力/速度）的原子算法。

### 1. 基础运算类
- **差值**:
  - 逻辑: `A - B`
  - 输入: 数值A, 数值B
  - 示例: `左右同步差 = 差值(A=左启动时间, B=右启动时间)`
- **和值**:
  - 逻辑: `A + B`
  - 输入: 数值A, 数值B
  - 示例: `总拉力 = 和值(A=左拉力, B=右拉力)`
- **乘积**:
  - 逻辑: `A * B`
  - 输入: 数值A, 数值B
  - 示例: `瞬时功率 = 乘积(A=实时拉力, B=实时速度)`
- **比率**:
  - 逻辑: `A / B`
  - 输入: 数值A, 数值B
  - 示例: `左右平衡度 = 比率(A=左拉力, B=右拉力)`
- **绝对值**:
  - 逻辑: `|A|`
  - 输入: 数值
  - 示例: `起点偏差绝对值 = 绝对值(数值=当前起点 - 初始起点)`

### 2. 统计分析类
- **极值**:
  - 逻辑: 窗口内或行程内的最大/最小值。
  - 输入: 数据源, 统计模式(最大/最小), 时间窗口
  - 示例: `向心峰值力 = 极值(数据=力, 模式=最大, 窗口=向心区间)`
- **均值**:
  - 逻辑: 窗口内或多次动作的平均值。
  - 输入: 数据源/历史数据, 样本量
  - 示例: `基准行程 = 均值(输入=历史前3次有效行程)`
- **波动率**:
  - 逻辑: `标准差 / 均值` (变异系数 CV)。用于评估动作稳定性。
  - 输入: 数据源, 时间窗口
  - 示例: `速度稳定性 = 波动率(数据=速度, 窗口=向心区间)`

### 3. 逻辑判定类
- **阈值判定**:
  - 逻辑: `数值 > 或 < 设定值`
  - 输入: 数值, 阈值, 模式(绝对/相对)
  - 特性: 支持绝对值(cm)与相对值(%)双模式。
- **趋势分析**:
  - 逻辑: 判定向心/离心/静止状态。
  - 输入: 位移数据, 时间
  - 参数: 抖动阈值(3-5cm), 有效阈值(>5cm)。
- **相似度**:
  - 逻辑: 比较当前行程波形与基准波形的重合度或数值接近度。
  - 输入: 当前波形, 基准波形
  - 示例: `行程相似度 = |A - B| / A`

## 数据池配置
- **原始数据** (传感器直出/驱动层):
  - `左/右电机位置`: 拉线长度 (cm)。(基础位移，精度 0.1cm)
  - `左/右电机拉力`: 实时阻力/拉力 (kg)。(基础负荷，精度 0.5kg)
  - `左/右电机速度`: 拉线速度 (m/s)。(向心为正，离心为负)
- **衍生数据** (计算合成):
  - `左/右行程`: `当前位置 - 起点位置` (cm)。(动作幅度判定)
  - `左/右加速度`: 速度变化率 (m/s²)。(反映爆发力与控制力)
  - `左/右加加速度 (Jerk)`: 加速度变化率 (m/s³)。(反映动作平滑度/控制力，数值越小动作越丝滑，用于检测颤抖/力竭)
  - `左/右瞬时功率`: `拉力 * 速度` (W)。(爆发力训练核心指标)
  - `左/右做功`: `拉力 * 位移` (J)。(动态训练容量计算，衡量做功总量)
  - `左/右冲量`: `拉力 * 时间` (N·s)。(静力训练负荷/耐力评估，弥补做功在静止时为0的缺陷)
  - `左/右力生成率 (RFD)`: 单位时间内的力变化量 (kg/s)。(神经募集能力/启动爆发力，检测"弹射起步"能力)
  - `左/右平均功率`: `做功 / 当前相位时长` (W)。(过程均值，反映持续做功效率，即"平均做功")
  - `左/右平均速度`: `行程 / 当前相位时长` (m/s)。(过程均值，VBT训练核心指标，用于判断是否力竭)
  - `左/右平均拉力`: `冲量 / 当前相位时长` (kg)。(过程均值，反映平均负荷强度)
  - `左/右峰值功率`: 当前相位内的最大功率 (W)。(过程极值，反映绝对爆发力上限)
  - `左/右峰值速度`: 当前相位内的最大速度 (m/s)。(过程极值，反映最大速度能力)
  - `左/右峰值拉力`: 当前相位内的最大拉力 (kg)。(过程极值，反映最大力量/粘滞点突破能力)
  - `左右行程偏差率`: `|左行程 - 右行程| / Max(左, 右)` (%)。(双侧同步性监测，消除身高/臂长差异影响)
  - `左右力偏差率`: `|左拉力 - 右拉力| / Max(左, 右)` (%)。(肌力平衡监测，消除负荷大小影响，通常 >15% 视为失衡)
  - `左右功率偏差率`: `|左功率 - 右功率| / Max(左, 右)` (%)。(爆发力平衡监测，高阶指标)
  - `左右RFD偏差率`: `|左RFD - 右RFD| / Max(左, 右)` (%)。(神经募集平衡监测，检测双侧启动爆发力差异)
  - `总拉力`: `左拉力 + 右拉力` (kg)。(总负荷监测，用于判断整体强度)
  - `总功率`: `左功率 + 右功率` (W)。(总代谢消耗/爆发力监测)
  - `总做功`: `左做功 + 右做功` (J)。(总能量消耗监测)
  - `总冲量`: `左冲量 + 右冲量` (N·s)。(总静力负荷监测)
- **系统数据**:
  - `时间戳`: 时间戳 (ms)
  - `当前计次`: 当前计次
  - `当前相位`: 当前相位 (准备/向心/顶峰/离心/复位)
  - `当前相位时长`: `当前时间 - 相位进入时间` (s)。(用于节奏控制/TUT监测，如"离心时长")
  - `电机状态`: `正常` | `过热` | `保护` | `故障`

## 指标池配置
- **说明**：基于数据源计算的二阶指标，供下游模块复用。
- **基本属性**：
  - *ID*：指标唯一标识 (如 `向心行程`, `启动时间差`)
  - *名称*：显示名称
  - *算法*：引用 [核心算子枚举]
  - *输入*：依赖的数据源或其他指标
  - *参数* (非必填)：算法参数
- **配置示例**：
  - *ID*: `向心行程`
  - *算法*: `差值` (向心终点位置 - 向心起点位置)
  - *ID*: `起点偏差`
  - *算法*: `差值` (当前向心起点 - 基准起点)

## 动态基准池配置 (自适应核心)
- **说明**：电机逻辑的核心，支持“初始化-锁定-修正”的动态闭环。
- **基本属性**：
  - *ID*：基准唯一标识 (如 `标准行程`)
  - *来源指标*：引用 [指标配置] ID
- **采集策略**：
  - **初始化**:
    - 条件: `邻近N次偏差 ≤ 阈值` (如 N=2, 偏差≤5cm)
    - 动作: 锁定均值为基准。
  - **自适应修正**:
    - 条件: `连续N次偏差 > 触发阈值 AND 偏差稳定性 ≤ 稳定阈值`
    - 动作: 更新基准为最新均值。
    - 补偿: `开启` (如: 触发修正时自动补录计次 +N)
    - 示例: 站位调整导致起点漂移，系统自动更新起点基准。

## 状态机配置
- **准备**:
  - 说明: 动作未开始或处于预备态。
  - 逻辑: 位移变化在 `±5cm` 以内。
  - *区间定义*:
    - **静止区**: `0 - 3cm` (忽略微小漂移)
    - **抖动区**: `3cm - 5cm` (视为预备/调整，不触发状态流转)
- **向心**:
  - 说明: 发力拉动阶段。
  - 触发: 位移变化 `> 5cm` (进入有效拉动区间)。
  - 属性: 记录启动时间戳、起点位置。
  - 示例: `位移 > 5cm`
- **顶峰**:
  - 说明: 动作顶点停顿状态 (顶峰收缩)。
  - 触发: 向心结束，速度趋近于 0 (或进入死区)，且位置处于行程高点。
  - 作用: 用于检测“顶峰收缩时长”或“静态保持”。
  - 示例: `速度 < 0.1m/s AND 位置 > 90%行程`
- **离心**:
  - 说明: 放回阶段。
  - 触发: 速度反向 (负向速度超过阈值)，位移明显回落。
  - 示例: `速度 < -0.1m/s`
- **复位**:
  - 说明: 动作结束回到起点。
  - 触发: 离心结束，位移回到起点附近 (如 `< 20% 行程`)。
  - 动作: 触发计次判定逻辑 (`向心过线` AND `离心过线`)。

## 有效性校验配置 (全局)
- **说明**: 定义动作计次的必要前置条件，过滤无效动作。
- **起点有效性**:
  - 逻辑: `|当前起点 - 基准起点| ≤ 10cm`
  - 作用: 防止用户大幅改变站位后，旧基准导致的误判。若不满足，不计次并触发基准修正检测。

## 单双侧判定逻辑
- **说明**：定义当前动作是单侧交替还是双侧同时，以及错误判定的合并逻辑。
- **模式识别**:
  - **探测期**: 左右启动时间差 `≤ 0.2s` (双侧) | `> 0.2s` (单侧)。
  - **锁定后**: 双侧模式下容差放宽至 `0.35s` (适应疲劳)。
- **错误触发逻辑** (双侧模式下):
  - *任一侧不达标*：只要有一侧数据异常即报错 (如左右手任一侧行程不足)。
  - *双侧均不达标*：两侧同时异常才报错。

## 错误判定配置
- **说明**：定义动作评价的时间维度判定逻辑。
- **枚举**：
  - **一票否决**: 只要检测到瞬间超标即判错 (如: 启动不同步)。
  - **达成即过**: 在特定区间内达到标准即判对 (如: 行程拉满)。
  - **实时跟随**: 状态随数据实时跳变 (如: 速度仪表盘)。

## 错误项配置架构
- **说明**：定义单个错误项的完整配置结构，包含识别逻辑与交互反馈。

### 1. 基本信息
- **标识**：
  - *ID*：唯一标识符 (如 `range`)
  - *名称*：显示名称 (如 `行程不足`)
- **属性**：
  - *优先级*：P0 (阻断) | P1 (核心) | P2 (优化)
  - *算法分类*：标准指标型 | 动力链型 | 节奏速率型

### 2. 核心算法配置 (多选一)

#### 类型A：标准指标型
- **适用场景**: 关注动作幅度是否达标、力量是否衰减。
- **属性定义**:
  - *目标指标*: 引用 [指标配置] ID。
  - *阈值*: 数值比较 (如 `> 80%`, `< 5cm`)。
  - *容忍度*: 定义不同难度等级下的判定阈值。
    - 格式：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *引导词* (非必填):
    - 格式：`条件 : 文案`
    - 动态阈值：使用 `{阈值}` 占位符代表当前生效的容忍度阈值。
    - 说明：支持多区间定义（含合格区间），系统根据实时指标值自动匹配触发。
- **配置示例**: 行程幅度
  - 目标指标: `向心行程`
  - 阈值: `< 80%`
  - 容忍度: `严格 < 85% / 标准 < 80% / 宽松 < 75%`
  - 引导词:
    - `< {阈值} : "拉满行程，不要偷懒"`
    - `>= {阈值} : "幅度完美，控制节奏缓慢下放"`

#### 类型B：动力链型
- **适用场景**: 关注双手配合是否默契、发力是否同步。
- **属性定义**:
  - *目标指标*: 引用 [指标配置] ID (如 `启动时间差`)。
  - *阈值*: 数值比较。
  - *容忍度*: `严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *引导词* (非必填):
    - 格式：`条件 : 文案`
    - 动态阈值：使用 `{阈值}` 占位符。
- **配置示例**: 左右同步
  - 目标指标: `启动时间差`
  - 阈值: `> 0.35s`
  - 容忍度: `严格 > 0.25s / 标准 > 0.35s / 宽松 > 0.45s`
  - 引导词:
    - `> {阈值} : "双手同时发力"`
    - `<= {阈值} : "节奏同步很好，保持住"`

#### 类型C：节奏速率型
- **适用场景**: 关注动作快慢节奏（如慢放快拉）。
- **属性定义**:
  - *目标指标*: 引用 [指标配置] ID (如 `离心时长`)。
  - *阈值*: 数值比较。
  - *容忍度*: `严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *引导词* (非必填):
    - 格式：`条件 : 文案`
- **配置示例**: 离心控制
  - 目标指标: `离心时长`
  - 阈值: `< 1.0s`
  - 容忍度: `严格 < 1.5s / 标准 < 1.0s / 宽松 < 0.5s`
  - 引导词:
    - `< {阈值} : "慢一点下放，控制离心"`
    - `>= {阈值} : "节奏控制很好，继续保持"`

#### 类型D：波形相似型
- **适用场景**: 关注发力曲线的平滑度或与标准模板的相似性。
- **属性定义**:
  - *目标指标*: 引用 [指标配置] ID (如 `行程相似度`)。
  - *参考波形*: `历史均值` | `标准模板`。
  - *阈值*: 相似度得分 (0.0 - 1.0)。
  - *容忍度*: `严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *引导词* (非必填):
    - 格式：`条件 : 文案`
- **配置示例**: 动作稳定性
  - 目标指标: `行程相似度`
  - 阈值: `< 0.8`
  - 引导词:
    - `< {阈值} : "动作不要变形，保持轨迹一致"`

### 3. 判定策略配置
- **检测时机**：
  - *枚举*：向心 | 离心 | 全程 | 启动瞬间 | 结束瞬间
  - *说明*：指定该错误项的激活时间窗口。
- **判定模式**：
  - *枚举*：一票否决 | 达成即过 | 实时跟随
- **时序防抖**：
  - *触发*：连续 N 次/帧触发。

### 4. 交互反馈配置
- **说明**：定义该错误触发时的视觉/听觉反馈。
- **设计原则**：采用 **“现状-差异-引导”** 的闭环设计。
- **状态：正确**：
  - *元素列表* (可配置多个)：
    - 类型：`提示词` | `特效` | `数值高亮`
    - 定义：内容 | 绑定指标
    - 样式：颜色（如绿色）、动效
    - **示例**：
      - `提示词`: 内容="幅度完美", 样式=绿色
      - `数值高亮`: 绑定=`行程`, 样式=绿色脉冲
- **状态：错误**：
  - *元素列表* (可配置多个)：
    - 类型：`提示词` | `仪表盘` | `波形图` | `数值`
    - 定义：内容 | 绑定指标
    - 样式：颜色（如红色/橙色）、闪烁
    - **示例**：
      - `提示词`: 内容="拉满行程", 样式=红色
      - `仪表盘`: 绑定=`速度`, 样式=红色刻度 (显示过快)
      - `波形图`: 绑定=`力曲线`, 样式=红色虚线 (显示掉力区域)
