# AI识别_动作纠错配置架构

## 全局逻辑控制
- **显示模式**：
  - 互斥（默认）：同一时刻仅显示一个最高优先级的纠错项。
  - 并行：同时显示所有触发的纠错项。
- **优先级排序**：
  - 升序（默认）：数值越小优先级越高 (P1 > P2)。
  - 降序：数值越大优先级越高。
- **优先级压制**：
  - 开启（默认）：当高优先级错误触发时，强制屏蔽低优先级错误的检测与显示。
  - 关闭：高优先级错误触发时，不影响低优先级错误的检测状态。
- **优先级挤占**：
  - 开启：允许新触发的错误挤占当前的纠错状态。
  - 关闭（默认）：一旦进入某种纠错状态，触发其他任何优先级的纠错都不挤占当前纠错，直到该纠错状态解除。
- **纠错交互触发频率**：
  - 配置数字（默认2）：决定几次错误触发纠错交互。

## 核心算子枚举
- **说明**：全系统通用的算法原子，用于虚拟点构建、基准采集及错误判定。

### 1. 几何构建类
- **中点**:
  - 逻辑: 计算输入点的几何中心。
  - 输入: 点列表 (如 `左髋, 右髋`)
  - 示例: `髋部中点 = 中点(左髋, 右髋)` (场景：深蹲-构建髋部中心以计算下蹲幅度和身体基准)
- **投影**:
  - 逻辑: 计算点在特定轴或平面上的投影。
  - 输入: `源` (提供非轴坐标), `参考` (提供轴坐标, 可选, 默认为源点)
  - 参数 (非必填): `方向` (垂直/水平), `偏移` (数值 | [点A, 点B]*系数, 可选)
  - 示例1: `左肘目标点 = 投影(源=左肘, 方向=垂直, 偏移=[左肘, 左腕]*-1.2)` (场景：推肩-构建小臂垂直的目标参考点)
  - 示例2: `左膝参考点 = 投影(源=左膝, 参考=左脚底中心, 方向=垂直)` (场景：深蹲-构建膝盖理想垂直投影点)
- **延伸**:
  - 逻辑: 从起点沿向量方向延伸指定长度。
  - 输入: `起点`, `方向` (向量/轴), `长度` (数值/参考段)
  - 示例: `左肩延伸点 = 延伸(起点=左肩, 方向=X轴负向, 长度=左大臂长)` (场景：侧平举-构建抬肘目标参考点)

### 2. 数值度量类
- **距离**:
  - 逻辑: 计算两点间距离。
  - 输入: 点A, 点B
  - 参数 (非必填): `模式` (欧氏/垂直/水平)
  - 示例: `髋膝垂直距离 = 距离(点A=髋中点, 点B=膝中点, 模式=垂直)` (场景：深蹲-测量下蹲时髋膝垂直落差)
- **角度**:
  - 逻辑: 计算几何夹角。
  - 输入: 点列表 (3点求内角, 2点求垂直/水平角)
  - 参数 (非必填): `模式` (内角/垂直/水平)
  - 示例: `大臂垂直角 = 角度(点列表=[左肩, 左肘], 模式=垂直)` (场景：侧平举-测量大臂抬起高度)
- **坐标**:
  - 逻辑: 获取点的坐标值。
  - 输入: 点
  - 参数 (非必填): `分量` (X/Y)
  - 示例: `髋部高度 = 坐标(点=髋中点, 分量=Y)` (场景：高抬腿-获取髋部高度基准)
- **点线距离**:
  - 逻辑: 计算点到直线的垂直距离。
  - 输入: 点, 线(起点-终点)
  - 示例: `弓背距离 = 点线距离(点=肩中点, 线=[耳中点, 髋中点])` (场景：俯身划船-测量背部拱起程度)

### 3. 逻辑判定类
- **压缩率**:
  - 逻辑: `当前度量值 / 动态基准值`
  - 示例: `下蹲深度压缩率 = 压缩率(当前=髋膝垂直距离, 基准=站立基准高度)` (场景：深蹲-判定下蹲幅度是否达标)
- **比例**:
  - 逻辑: `分子度量值 / 分母度量值`
  - 示例: `膝内扣比例 = 比例(分子=双膝间距, 分母=双髋间距)` (场景：深蹲-判定膝盖是否内扣)
- **偏移率**:
  - 逻辑: `点线距离 / 参考长度`
  - 示例: `侧倾偏移率 = 偏移率(距离=肩髋水平偏移, 参考=躯干长度)` (场景：弓步-判定躯干是否侧倾)
- **波动**:
  - 逻辑: `(最大值 - 最小值) / 参考值` (在时间窗口内)
  - 示例: `躯干晃动率 = 波动(数值=躯干垂直角, 窗口=5帧)` (场景：二头弯举-判定躯干是否借力晃动)
- **阈值**:
  - 逻辑: `数值 > 或 < 设定值`
  - 示例: `抬肘达标 = 阈值(数值=大臂垂直角, 设定="> 80°")` (场景：侧平举-判定抬肘高度是否达标)

## 虚拟计算点池配置
- **类型**：原生骨骼点 | 虚拟计算点
- **原生骨骼点**：
  - ID：模型原始ID
  - 名称：标准骨骼名称（如 `nose`, `left_shoulder`）
- **虚拟计算点**：
  - ID：自定义唯一ID (>100)
  - 名称：自定义名称 (如 `髋部中点`)
  - *算法*：引用 [核心算子枚举-几何构建类]
  - *输入*：依赖的点ID列表
  - *参数* (非必填)：算法所需的额外参数

## 指标池配置
- **说明**：统一通过核心算子定义所有需要计算的物理量，供下游模块复用。
- **基本属性**：
  - *ID*：指标唯一标识 (如 `膝关节角度`, `髋高比`)
  - *名称*：显示名称 (如 `膝关节角度`)
  - *算法*：引用 [核心算子枚举-数值度量类/逻辑判定类]
  - *输入*：依赖的点ID 或 其他指标ID
  - *参数* (非必填)：算法参数
- **配置示例**：
  - ID: `大臂垂直角`
  - 算法: `角度`
  - 输入: `[左肩, 左肘]`
  - 参数: `模式=垂直`

## 动态基准池配置 (非必填)
- **说明**：定义用于计算压缩率或相对位置的参考基准值
- **基本属性**：
  - *ID*：基准唯一标识 (如 `站立基准`)
  - *名称*：显示名称 (如 `站立基准高度`)
  - *来源指标*：引用 [指标配置] 中的ID (如 `髋膝垂直距离`)
- **采集逻辑**：
  - 算法：`智能基准校准`
  - 机制：自动在`准备`状态下采集最大值，并应用阻尼抗噪和微量衰减
  - 参数：
    - 阻尼：默认 0.05 (抗噪)
    - 衰减：默认 0.9995 (防止虚高)

## 状态机配置
- **准备**：
  - 说明：动作就绪状态，**允许采集动态基准值**
  - 逻辑：系统初始化或从复位状态判定稳定后进入
  - 属性：基准校准开关
  - 示例：`进入: 复位状态保持 > 0.5s`
- **向心**：
  - 说明：动作开始发力或下放的阶段
  - 定义：触发条件（引用指标ID + 运算器 + 阈值）
  - 示例：`下蹲深度 ≤ 85%` (深蹲开始)
- **顶峰**:
  - 说明：动作最高点或发力顶峰状态 (衔接向心与离心)，用于检测顶峰收缩。
  - 定义：触发条件（位置进入极值区间 + 速度低于阈值 + 持续时长）。
  - 策略：推荐使用“位置区间”而非纯速度，以抵抗视觉抖动。
  - 示例：`下蹲深度 < 10% AND 停留时长 > 0.2s`
- **进入纠错区间 (非必填)**：
  - 说明：进入核心判定区域，开始激活“过程修正”或“达成即过”逻辑
  - 定义：触发条件（引用指标ID + 运算器 + 阈值）
  - 示例：`下蹲深度 ≤ 30%` (进入深蹲底部)
- **离开纠错区间 (非必填)**：
  - 说明：离开核心判定区域，锁定“过程修正”结果
  - 定义：触发条件（引用指标ID + 运算器 + 阈值）
  - 示例：`下蹲深度 ＞ 30%` (离开深蹲底部)
- **离心**：
  - 说明：动作还原或起身的运动过程
  - 定义：触发条件（引用指标ID + 运算器 + 阈值）
  - 示例：`下蹲深度 ＞ 30%` (开始起身)
- **复位**：
  - 说明：动作结束后的过渡状态，**暂停基准采集**，防止动作惯性干扰
  - 定义：触发条件（引用指标ID + 运算器 + 阈值）
  - 示例：`进入: 下蹲深度 ≥ 92% (动作完成)`
- **保持 (非必填)**：
  - 说明：动作过程中的静力保持阶段
  - 定义：进入条件 + 退出条件 + 最小保持时长
  - 示例：`进入: 下蹲深度 < 15%; 退出: 下蹲深度 > 20%; 时长: 1s`
- **超时熔断 (非必填)**：
  - 说明：异常状态下的强制复位机制
  - 定义：超时阈值（秒） + 状态校验指标
  - 逻辑：若 (当前状态时长 > 超时阈值) 且 (指标满足复位条件) 则 强制转为准备态
  - 示例：`时长 > 6s AND 下蹲深度 > 75%` (深蹲卡死复位)

## 错误项配置架构
- **说明**：定义单个错误项的完整配置结构，包含识别逻辑与交互反馈。

### 1. 基本信息
- **标识**：
  - *ID*：唯一标识符 (如 `valgus`)
  - *名称*：显示名称 (如 `膝内扣`)
  - *描述*：详细说明 (如 `检测下蹲过程中双膝是否内扣`)
- **属性**：
  - *优先级*：P0 (阻断) | P1 (核心) | P2 (优化)
  - *算法分类*：`标准指标型` | `动力链型` | `轨迹拟合型` | `节奏速率型` | `姿态相似型`

### 2. 核心算法配置 (多选一)
#### 类型A：标准指标型
- **适用场景**：关注单一时刻的姿态标准（如：蹲得够不够深、手举得够不够高、身体有没有歪）。
- **属性定义**：
  - *目标指标*：引用 [指标配置] ID (支持单个或多个)。
  - *阈值*：数值比较 (如 `> 80`, `< 0.1`)，与指标一一对应。
  - *组合逻辑*：`AND` (同时满足) | `OR` (任一满足) (仅多指标时有效)。
  - *容忍度*：定义不同难度等级下的判定阈值。
    - 格式：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *计算变量*：记为 `单指标` 或 `复合指标`。
  - *引导词* (非必填)：定义指标数值区间对应的实时反馈。
    - 格式：`条件 : 文案`
    - 动态阈值：使用 `{阈值}` 占位符代表当前生效的容忍度阈值。
    - 说明：支持多区间定义（含合格区间），系统根据实时指标值自动匹配触发。
- **配置示例1**：侧平举-抬肘高度 (单指标)
  - 目标指标：`大臂垂直角`
  - 阈值：`[80°, 100°]`
  - 容忍度：`严格 [85°, 95°] / 标准 [80°, 100°] / 宽松 [70°, 110°]`
  - 引导词:
    - `单指标 < {阈值_min} : "继续向上抬肘，寻找发力感"`
    - `单指标 ∈ [{阈值_min}, {阈值_max}] : "高度完美，控制节奏缓慢下放"`
    - `单指标 > {阈值_max} : "略微下放，与肩同高即可"`
  - *纠错价值*：确保三角肌中束充分收缩，避免因抬起高度不足导致的训练效果打折。
- **配置示例2**：站姿推举-腰椎反弓 (复合指标)
  - 目标指标：`[躯干后仰角, 髋部前移量]`
  - 阈值：`[> 15°, > 0.1]`
  - 组合逻辑：`AND` (同时满足后仰且顶髋，判定为反弓代偿)
  - 容忍度：`严格 [> 10°, > 0.05] / 标准 [> 15°, > 0.1] / 宽松 [> 20°, > 0.15]`
  - 引导词:
    - `复合指标 满足 {阈值} : "收紧核心，不要过度反弓腰背"`
    - `复合指标 不满足 {阈值} : "核心收得很紧，脊柱中立位保持得很好"`
  - *纠错价值*：防止用户为了推起更大重量而过度伸展腰椎，导致下背部受伤风险。

#### 类型B：动力链型
- **适用场景**：关注身体各部位的配合默契度（如：手脚是否协调、发力传导是否顺畅）。
- **枚举**：同步型 | 顺序型

**B1. 同步型**
- **场景**：关注动作的同步性（如：膝盖和髋部是否同时伸展，避免脱节）。
- **属性定义**：
  - *基准指标*：引用 [指标配置] ID (自变量)。
  - *跟随指标*：引用 [指标配置] ID (因变量)。
  - *判定公式*：`偏差 = | 跟随指标 - (基准指标 * 系数 + 偏移) | ≤ 容忍阈值`
  - *容忍阈值*：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *引导词* (非必填)：
    - 格式：`条件 : 文案`
    - 动态阈值：使用 `{阈值}` 占位符代表当前生效的容忍阈值。
- **配置示例1**：硬拉-膝髋同步 (1:1 协同)
  - 基准指标：`髋伸直角`
  - 跟随指标：`膝伸直角`
  - 判定公式：`偏差 = | 膝伸直角 - (髋伸直角 * 1.0 + 0) | ≤ {阈值}` (释义：膝关节与髋关节应保持 1:1 同步伸展)
  - 容忍阈值：`严格 ≤ 10° / 标准 ≤ 15° / 宽松 ≤ 20°`
  - 引导词:
    - `偏差 > {阈值} : "膝髋脱节了，注意同步伸展"`
    - `偏差 <= {阈值} : "协同发力感很好，保持这个节奏"`
  - *纠错价值*：防止“先伸膝后伸髋”（早安式硬拉），保护腰椎并确保臀大肌正确发力。
- **配置示例2**：侧平举-沉肩定型 (动力链解耦)
  - 基准指标：`大臂垂直角` (自变量变化)
  - 跟随指标：`肩耳距离` (因变量保持)
  - 判定公式：`偏差 = | 肩耳距离 - (大臂垂直角 * 0 + 初始距离) | / 初始距离 ≤ {阈值}` (释义：无论手臂如何抬起，肩部始终保持下沉，不向耳部耸起)
  - 容忍阈值：`严格 ≤ 0.10 / 标准 ≤ 0.15 / 宽松 ≤ 0.20` (即允许 10%~20% 的耸肩幅度)
  - 引导词:
    - `偏差 > {阈值} : "肩膀沉下去，不要耸肩借力"`
    - `偏差 <= {阈值} : "沉肩状态保持得很好，三角肌受力充分"`
  - *纠错价值*：防止斜方肌上束过度代偿（耸肩），确保三角肌中束孤立发力，避免颈部紧张。

**B2. 顺序型**
- **场景**：关注动作衔接的时序关系（如：力量传递是否连贯，是否存在过早发力或动力脱节）。
- **属性定义**：
  - *后置指标*：引用 [指标配置] ID (下游动作)。
  - *前置指标*：引用 [指标配置] ID (上游动作)。
  - *判定公式*：`重叠时长 = (前置完成时刻 - 后置开始时刻) ≤ 容忍时长`
  - *阈值设定*：定义判定动作“开始”与“完成”的临界值。
  - *容忍时长*：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *引导词* (非必填)：
    - 格式：`条件 : 文案`
    - 动态阈值：使用 `{阈值}` 占位符代表当前生效的容忍时长。
- **配置示例1**：借力推举-过早推臂 (推类协同)
  - 后置指标：`肘伸直角` (开始阈值 `> 60°`)
  - 前置指标：`髋伸直角` (完成阈值 `> 160°`)
  - 判定公式：`重叠时长 = (髋伸直角完成时刻 - 肘伸直角开始时刻) ≤ {阈值}` (释义：手臂开始推起时刻 必须晚于 髋部伸展时刻)
  - 容忍时长：`严格 0.05s / 标准 0.1s / 宽松 0.15s`
  - 引导词:
    - `重叠时长 > {阈值} : "髋部顶出去，借着冲力向上推"`
    - `重叠时长 <= {阈值} : "推举时机完美，爆发力传导充分"`
  - *纠错价值*：借力推举的核心在于利用伸髋的惯性将重量“顶”起来。若手臂过早推起，就变成了纯肩部推举，失去了借力的意义。
- **配置示例2**：深蹲划船-过早拉臂 (拉类协同)
  - 后置指标：`肘屈曲角` (开始阈值 `< 120°`)
  - 前置指标：`膝伸直角` (完成阈值 `> 160°`)
  - 判定公式：`重叠时长 = (膝伸直角完成时刻 - 肘屈曲角开始时刻) ≤ {阈值}` (释义：手臂开始发力时刻 必须晚于 腿部蹬直时刻)
  - 容忍时长：`严格 0.05s / 标准 0.1s / 宽松 0.15s`
  - 引导词:
    - `重叠时长 > {阈值} : "先蹬腿站直，再向后拉臂"`
    - `重叠时长 <= {阈值} : "下肢驱动有力，衔接很顺畅"`
  - *纠错价值*：深蹲划船要求利用蹬腿的爆发力带动上肢拉动。过早拉臂会造成腰部压力过大且发力效率低下。

#### 类型C：轨迹拟合型
- **适用场景**：关注动作过程中的运动路线（如：器械轨迹是否标准、身体重心移动是否稳定）。
- **核心逻辑**：基于“管状约束”或“模板比对”计算全程拟合度，并支持分段纠错。
- **枚举**：直线型 | 弧线型 | 模板轨迹型

**C1. 直线型**
- **场景**：关注走直线的动作（如：深蹲重心是否直上直下、推举轨迹是否垂直）。
- **属性定义**：
  - *追踪点*：引用 [虚拟计算点] ID (需检测轨迹的动点)。
  - *参考点*：引用 [虚拟计算点] ID (相对静止的参考点，用于构建坐标系)。
  - *理想轴线*：`垂直` | `水平` | `自定义向量` (如：动作起点至终点的连线)。
  - *动态基准*：引用 [动态基准配置] ID (用于计算相对偏差的参考长度)。
  - *判定公式*：`拟合度 = (合格帧数 / 总帧数) ≥ 达标阈值`
  - *单帧判定*：`偏差 = (点线距离 / 动态基准) ≤ {容忍半径}`
  - *容忍半径*：定义虚拟管道的允许宽度（归一化比例）。
    - 格式：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *达标阈值*：定义全程合格所需的拟合度百分比。
    - 格式：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *分段诊断* (非必填)：
    - 逻辑：将轨迹按行程进度(0%-100%)分为多段，回溯判断错误发生的主要区间。
    - 格式：`区间 : 引导词` (如 `[0, 0.3] : "起步阶段重心不稳"`)
- **配置示例1**：深蹲-重心垂直度 (垂直稳定)
  - 追踪点：`髋部中点`
  - 参考点：`脚底中心` (地面投影)
  - 理想轴线：`垂直`
  - 动态基准：`躯干长度基准`
  - 容忍半径：`严格 < 0.08 / 标准 < 0.10 / 宽松 < 0.12`
  - 达标阈值：`严格 > 90% / 标准 > 85% / 宽松 > 80%`
  - 分段诊断:
    - `[0.0, 0.3] : "下蹲初期重心偏离，注意屈髋节奏"`
    - `[0.3, 0.8] : "下蹲过程重心不稳，压实全脚掌"`
    - `[0.8, 1.0] : "底部重心偏移，注意脚踝灵活性"`
  - *纠错价值*：深蹲时重心若偏离足中，会导致膝关节剪切力增加或身体失去平衡。
- **配置示例2**：卧推-推起轨迹 (斜向向量)
  - 追踪点：`腕关节`
  - 参考点：`肩关节`
  - 理想轴线：`自定义向量` (起点=胸部落点, 终点=肩部上方)
  - 动态基准：`最大臂长基准`
  - 容忍半径：`严格 < 0.03 / 标准 < 0.05 / 宽松 < 0.08`
  - 达标阈值：`严格 > 95% / 标准 > 90% / 宽松 > 85%`
  - 分段诊断:
    - `[0.0, 0.4] : "推起初期轨迹偏离，胸肌发力不足"`
    - `[0.4, 1.0] : "推起后半程轨迹不稳，控制住杠铃"`
  - *纠错价值*：卧推的最佳轨迹并非垂直直上直下，而是略带弧度（J形），此轨迹能最大化胸肌发力并保护肩关节。

**C2. 弧线型**
- **场景**：关注画圆弧的动作（如：侧平举时手臂是否伸得够远、飞鸟时是否抱桶定型）。
- **属性定义**：
  - *追踪点*：引用 [虚拟计算点] ID (做圆周运动的动点)。
  - *圆心点*：引用 [虚拟计算点] ID (旋转轴心点)。
  - *基准半径*：引用 [动态基准配置] ID (理想的旋转半径长度)。
  - *判定公式*：`拟合度 = (合格帧数 / 总帧数) ≥ 达标阈值`
  - *单帧判定*：`偏差 = (|实时半径 - 基准半径| / 基准半径) ≤ {容忍半径}`
  - *容忍半径*：定义虚拟球壳的允许厚度（归一化比例）。
    - 格式：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *达标阈值*：定义全程合格所需的拟合度百分比。
    - 格式：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *分段诊断* (非必填)：
    - 格式：`区间 : 引导词`
- **配置示例1**：侧平举-手臂伸展度 (防止屈肘偷懒)
  - 追踪点：`腕关节`
  - 圆心点：`肩关节`
  - 基准半径：`最大臂长基准`
  - 容忍半径：`严格 < 0.10 / 标准 < 0.15 / 宽松 < 0.20`
  - 达标阈值：`严格 > 90% / 标准 > 85% / 宽松 > 80%`
  - 分段诊断:
    - `[0.5, 1.0] : "顶峰阶段不要屈肘，保持力臂"`
  - *纠错价值*：防止用户在力竭时通过过度屈肘来缩短力臂，降低训练强度。
- **配置示例2**：哑铃飞鸟-抱桶定型 (力臂保持)
  - 追踪点：`腕关节`
  - 圆心点：`肩关节`
  - 基准半径：`准备态臂长基准` (微屈固定)
  - 容忍半径：`严格 < 0.08 / 标准 < 0.10 / 宽松 < 0.12`
  - 达标阈值：`严格 > 90% / 标准 > 85% / 宽松 > 80%`
  - 分段诊断:
    - `[0.0, 0.5] : "下放到底部时，注意肘部不要弯曲"`
  - *纠错价值*：飞鸟动作要求肘关节角度固定，若变为推举（半径变短），则失去了孤立胸肌的意义。

**C3. 模板轨迹型**
- **场景**：关注特定技巧轨迹（如：抓举时杠铃是否贴身S型上升）。
- **属性定义**：
  - *追踪点*：引用 [虚拟计算点] ID (需检测轨迹的动点)。
  - *参考模板*：引用标准轨迹数据ID (归一化的理想路径)。
  - *比对策略*：`动态规整` (关注时序形状) | `路径距离` (关注空间重合度)。
  - *判定公式*：`拟合度 = (轨迹匹配得分) ≥ 达标阈值`
  - *达标阈值*：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *缩放适配*：`开启` | `关闭` (是否自动适配用户肢体长度/动作幅度)。
  - *分段诊断* (非必填)：
    - 逻辑：基于DTW对齐后的分段匹配得分进行诊断。
    - 格式：`区间 : 引导词`
- **配置示例1**：抓举-杠铃轨迹 (S型发力)
  - 追踪点：`腕关节` (代表杠铃位置)
  - 参考模板：`snatch_bar_path_std`
  - 比对策略：`动态规整`
  - 达标阈值：`严格 > 0.90 / 标准 > 0.85 / 宽松 > 0.80`
  - 分段诊断:
    - `[0.0, 0.3] : "一拉阶段杠铃要贴近小腿"`
    - `[0.3, 0.6] : "二拉发力时，杠铃不要撞击髋部"`
    - `[0.6, 1.0] : "下钻支撑时，轨迹要垂直锁定"`
  - *纠错价值*：抓举的杠铃轨迹应贴近身体呈S型上升。若轨迹远离身体，将产生巨大的力矩，导致动作失败或腰部受伤。

#### 类型D：节奏速率型
- **适用场景**：关注动作的时间与强度特征（如：离心控制时长、向心爆发速度、顶峰静力保持、多次动作节奏稳定性）。
- **属性定义**：
  - *目标指标*：引用 [指标配置] ID。
  - *分析维度*：`时长` | `速度` | `加速度`。
  - *计算模式*：`均值` | `峰值`。
    - 说明：仅适用于速度/加速度等时序数据。时长维度默认忽略此项。
    - 均值：反映整体抗阻能力（如肌肥大训练）。
    - 峰值：反映瞬间爆发能力（如功率训练）。
  - *监测阶段*：`向心` | `离心` | `全程` | `静力保持`。
  - *评价类型*：`绝对值` (默认) | `一致性` 。
    - 绝对值：关注单次动作的数值表现。
    - 一致性：关注多次动作的数值波动情况。
  - *参考窗口*：定义计算一致性的历史窗口大小 (仅一致性模式有效，如 `3次`)。
  - *阈值条件*：数值比较条件。
    - 格式：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *计算变量*：
    - `监测值`：单次动作的计算结果 (绝对值模式)。
    - `波动率`：基于参考窗口计算的变异系数 CV (标准差/均值) (一致性模式)。
  - *引导词* (非必填)：
    - 格式：`条件 : 文案`
    - 动态阈值：使用 `{阈值}` 占位符代表当前生效的阈值条件。
- **配置示例1**：深蹲-离心慢放
  - 目标指标：`下蹲深度`
  - 分析维度：`时长`
  - 监测阶段：`离心`
  - 阈值条件：`严格 > 4.0s / 标准 > 3.0s / 宽松 > 2.0s`
  - 引导词:
    - `监测值 < {阈值} : "下放太快了，慢一点感受肌肉拉伸"`
    - `监测值 >= {阈值} : "离心控制完美，就是这个节奏"`
  - *纠错价值*：离心阶段是肌肉破坏和增长的关键。控制下放速度能显著提升肌肥大效果，防止借力下坠。
- **配置示例2**：高翻-爆发力
  - 目标指标：`髋部高度`
  - 分析维度：`速度`
  - 计算模式：`峰值`
  - 监测阶段：`向心`
  - 阈值条件：`严格 > 1.8 / 标准 > 1.5 / 宽松 > 1.2` (m/s)
  - 引导词:
    - `监测值 < {阈值} : "拉起速度太慢，爆发力出来！"`
    - `监测值 >= {阈值} : "起势迅猛，爆发力很强"`
  - *纠错价值*：高翻是典型的功率训练。若拉起速度不够，说明爆发力不足，动作将退化为普通的硬拉+提拉。
- **配置示例3**：暂停深蹲-底部停顿 (静力控制)
  - 目标指标：`下蹲深度`
  - 分析维度：`时长`
  - 监测阶段：`静力保持` (需状态机定义"保持"状态)
  - 阈值条件：`严格 > 1.0s / 标准 > 0.5s / 宽松 > 0.0s`
  - 引导词:
    - `监测值 < {阈值} : "底部停顿一下，不要利用惯性弹起"`
    - `监测值 >= {阈值} : "停顿控制很好，起！"`
  - *纠错价值*：暂停深蹲要求在底部消除牵张反射（惯性），纯靠肌肉力量启动。检测停顿时间能防止用户“弹”起来。
- **配置示例4**：卧推-节奏稳定性 (一致性监测)
  - 目标指标：`推举高度`
  - 分析维度：`时长`
  - 监测阶段：`离心` (下放阶段)
  - 评价类型：`一致性`
  - 参考窗口：`3次`
  - 阈值条件：`严格 < 0.10 / 标准 < 0.15 / 宽松 < 0.20` (变异系数 CV)
  - 引导词:
    - `波动率 > {阈值} : "保持节奏稳定，不要忽快忽慢"`
    - `波动率 <= {阈值} : "节奏控制非常稳定"`
  - *纠错价值*：高级训练者需要保持每一次动作的节奏高度一致（如 3-0-1-0），忽快忽慢意味着神经控制疲劳或注意力不集中。

#### 类型E：姿态相似型
- **适用场景**：关注静态动作的整体标准度（如：瑜伽体式是否到位、平板支撑是否像一块钢板）。
- **核心逻辑**：将身体姿态抽象为多维向量，计算与标准模版的整体相似性。
- **属性定义**：
  - *比对指标集*：定义参与比对的物理量及其重要性权重。
    - 格式：`[指标ID : 权重, ...]`
    - 说明：权重为相对值 (如 1.0, 0.5)，系统自动归一化。未填权重默认为 1.0。
  - *参考模板*：引用已录制的标准动作数据ID。
  - *比对维度*：
    - **形态**：输入为 [角度类] 指标，关注几何形状 (推荐，不受身高影响)。
    - **位置**：输入为 [归一化坐标] 指标，关注相对空间位置。
  - *判定公式*：`相似度 = (加权匹配得分) ≥ 达标阈值`
  - *达标阈值*：相似度得分下限 (0.0 - 1.0，越接近1.0越标准)。
    - 格式：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
  - *引导词* (非必填)：
    - 格式：`条件 : 文案`
    - 动态阈值：使用 `{阈值}` 占位符代表当前生效的达标阈值。
- **配置示例1**：瑜伽-树式 (静态平衡)
  - 比对指标集：`[左肩展角: 1.0, 右肩展角: 1.0, 髋角: 1.5]` (髋部平衡是核心)
  - 参考模板：`yoga_tree_std`
  - 比对维度：`形态`
  - 达标阈值：`严格 > 0.90 / 标准 > 0.85 / 宽松 > 0.80`
  - 引导词:
    - `相似度 < {阈值} : "身体晃动了，沉肩坠肘找平衡"`
    - `相似度 >= {阈值} : "体态非常优美，保持呼吸平稳"`
  - *纠错价值*：瑜伽体式讲究整体的几何美感与平衡。通过相似度比对，能纠正用户细微的体态偏差（如髋部不正、肩部耸起）。
- **配置示例2**：平板支撑-标准体态 (核心稳定)
  - 比对指标集：`[头肩髋膝踝五点共线: 2.0, 大臂垂直角: 1.0]` (核心直线最重要)
  - 参考模板：`plank_std`
  - 比对维度：`形态`
  - 达标阈值：`严格 > 0.95 / 标准 > 0.90 / 宽松 > 0.85`
  - 引导词:
    - `相似度 < {阈值} : "腰塌了，收紧核心把身体顶直"`
    - `相似度 >= {阈值} : "像钢板一样直，核心非常稳定"`
  - *纠错价值*：平板支撑要求身体呈一条直线。相似度算法能综合判断塌腰、撅臀、低头等多种错误，比单一角度阈值更准确。

### 3. 判定策略配置
- **说明**：定义从“指标计算”到“最终判定”的数据处理流。
- **检测时机**：
  - *枚举*：`全程` (默认) | `向心` | `离心` | `静力保持` | `纠错区间`
  - *说明*：指定该错误项的激活时间窗口。
  - *智能回退*：若指定为 `纠错区间` 但状态机未定义，则自动回退为 `全程`。

- **判定模式**：
  - *枚举*：`实时跟随` (默认) | `一票否决` | `达成即过` | `过程修正`
  - *说明*：选择适用的时间维度判定逻辑（详见全局定义）。
  - *特殊说明*：对于`轨迹拟合型`，最终考核由拟合度标准决定，此模式主要控制交互反馈的锁定与解除逻辑。
  - **一票否决**：
    - 逻辑：`最终结果 = 合格 IF (检测时机内 无 任何帧不达标)`
    - 触发：初始默认为合格，一旦检测到错误（满足防抖），即刻锁定为不合格。
    - 解除：本轮不可解除，需等待下一轮动作重置。
    - 场景：动作标准类，需要全程保持正确的动作。
    - 示例：小臂不垂直（推肩）；耸肩（推肩）。
  - **达成即过**：
    - 逻辑：`最终结果 = 合格 IF (检测时机内 存在 任意帧达标)`
    - 触发：初始默认为不合格，只要在检测时机内达标一次，即刻锁定为合格。
    - 解除：达标即解除错误，保持合格直至本轮结束。
    - 场景：动作幅度类，需要特定关键帧区间动作幅度达标的动作。
    - 示例：下蹲深度不足（深蹲）、抬肘高度不足（侧平举）。
  - **过程修正**：
    - 逻辑：`最终结果 = 合格 IF (检测时机结束时 状态达标)`
    - 触发：进入检测时机时若不达标，判为不合格。
    - 解除：必须在检测时机结束前修正动作（达标），并保持至时机结束。
    - 场景：动作标准类，允许用户在过程中自我纠正。
    - 示例：膝内扣（深蹲）；膝内扣（弓步）。
  - **实时跟随**：
    - 逻辑：`最终结果 = 当前帧状态` (无状态记忆)
    - 触发/解除：所见即所得，状态随每一帧检测结果实时跳变。
    - 场景：辅助参考线、非关键性的软性提示。
    - 示例：脊柱曲率实时显示、重心投影点跟随。

- **单双侧逻辑**：
  - *模式*：`整体` (默认) | `分侧`
  - *触发逻辑*：`任一侧不达标` (默认) | `双侧均不达标` (仅分侧模式有效)
  - *解除逻辑*：`双侧均达标` (默认) | `任一侧达标` (仅分侧模式有效)
  - **整体**：
    - 逻辑：`指标值不达标`
    - 说明：针对躯干、双肢协同或单一整体指标的判定，不区分左右侧。
    - 示例：`躯干侧倾 OR 深蹲下蹲幅度不足`
  - **触发逻辑详解**：
    - *任一侧不达标*：
      - 逻辑：`左侧不达标 OR 右侧不达标`
      - 说明：只要有一侧不达标，即触发错误（严判，适用于安全类指标）。
      - 示例：`左膝内扣 OR 右膝内扣 → 触发错误`
    - *双侧均不达标*：
      - 逻辑：`左侧不达标 AND 右侧不达标`
      - 说明：两侧同时不达标时，才触发错误（宽判）。
  - **解除逻辑详解**：
    - *双侧均达标*：
      - 逻辑：`左侧达标 AND 右侧达标`
      - 说明：必须两侧都达标，错误才解除（严进严出，推荐）。
      - 示例：`左膝正常 AND 右膝正常 → 解除错误`
    - *任一侧达标*：
      - 逻辑：`左侧达标 OR 右侧达标`
      - 说明：只要有一侧达标，错误即解除（宽进宽出）。

- **时序防抖**：
  - *触发帧数*：连续 N 帧满足条件才触发错误（默认 3帧）。
  - *解除帧数*：连续 N 帧不满足条件才解除错误（默认 3帧）。
