# AI识别_动作纠错配置架构

## 全局逻辑控制
- **显示模式**：
  - 互斥（默认）：同一时刻仅显示一个最高优先级的纠错项。
  - 并行：同时显示所有触发的纠错项。
- **优先级排序**：
  - 升序（默认）：数值越小优先级越高 (P1 > P2)。
  - 降序：数值越大优先级越高。
- **优先级压制**：
  - 开启（默认）：当高优先级错误触发时，强制屏蔽低优先级错误的检测与显示。
  - 关闭：高优先级错误触发时，不影响低优先级错误的检测状态。
- **优先级挤占**：
  - 开启：允许新触发的错误挤占当前的纠错状态。
  - 关闭（默认）：一旦进入某种纠错状态，触发其他任何优先级的纠错都不挤占当前纠错，直到该纠错状态解除。
- **纠错交互触发频率**：
  - 配置数字（默认2）：决定几次错误触发纠错交互。

## 核心算子枚举
- **说明**：全系统通用的算法原子，用于虚拟点构建、基准采集及错误判定。

### 1. 几何构建类
- **中点**:
  - 逻辑: 计算输入点的几何中心。
  - 输入: 点列表 (如 `左髋, 右髋`)
  - 示例: `髋部中点 = 中点(左髋, 右髋)` (场景：深蹲-构建髋部中心以计算下蹲幅度和身体基准)
- **投影**:
  - 逻辑: 计算点在特定轴或平面上的投影。
  - 输入: `源` (提供非轴坐标), `参考` (提供轴坐标, 可选, 默认为源点)
  - 参数: `方向` (垂直/水平), `偏移` (数值 | [点A, 点B]*系数, 可选)
  - 示例1: `左肘目标点 = 投影(源=左肘, 方向=垂直, 偏移=[左肘, 左腕]*-1.2)` (场景：推肩-构建小臂垂直的目标参考点)
  - 示例2: `左膝参考点 = 投影(源=左膝, 参考=左脚底中心, 方向=垂直)` (场景：深蹲-构建膝盖理想垂直投影点)
- **延伸**:
  - 逻辑: 从起点沿向量方向延伸指定长度。
  - 输入: `起点`, `方向` (向量/轴), `长度` (数值/参考段)
  - 示例: `左肩延伸点 = 延伸(起点=左肩, 方向=X轴负向, 长度=左大臂长)` (场景：侧平举-构建抬肘目标参考点)

### 2. 数值度量类
- **距离**:
  - 逻辑: 计算两点间距离。
  - 输入: 点A, 点B
  - 参数: `模式` (欧氏/垂直/水平)
  - 示例: `髋膝垂直距离 = 距离(点A=髋中点, 点B=膝中点, 模式=垂直)` (场景：深蹲-测量下蹲时髋膝垂直落差)
- **角度**:
  - 逻辑: 计算几何夹角。
  - 输入: 点列表 (3点求内角, 2点求垂直/水平角)
  - 参数: `模式` (内角/垂直/水平)
  - 示例: `大臂垂直角 = 角度(点列表=[左肩, 左肘], 模式=垂直)` (场景：侧平举-测量大臂抬起高度)
- **坐标**:
  - 逻辑: 获取点的坐标值。
  - 输入: 点
  - 参数: `分量` (X/Y)
  - 示例: `髋部高度 = 坐标(点=髋中点, 分量=Y)` (场景：高抬腿-获取髋部高度基准)
- **点线距离**:
  - 逻辑: 计算点到直线的垂直距离。
  - 输入: 点, 线(起点-终点)
  - 示例: `弓背距离 = 点线距离(点=肩中点, 线=[耳中点, 髋中点])` (场景：俯身划船-测量背部拱起程度)

### 3. 逻辑判定类
- **压缩率**:
  - 逻辑: `当前度量值 / 动态基准值`
  - 示例: `下蹲深度压缩率 = 压缩率(当前=髋膝垂直距离, 基准=站立基准高度)` (场景：深蹲-判定下蹲幅度是否达标)
- **比例**:
  - 逻辑: `分子度量值 / 分母度量值`
  - 示例: `膝内扣比例 = 比例(分子=双膝间距, 分母=双髋间距)` (场景：深蹲-判定膝盖是否内扣)
- **偏移率**:
  - 逻辑: `点线距离 / 参考长度`
  - 示例: `侧倾偏移率 = 偏移率(距离=肩髋水平偏移, 参考=躯干长度)` (场景：弓步-判定躯干是否侧倾)
- **波动**:
  - 逻辑: `(最大值 - 最小值) / 参考值` (在时间窗口内)
  - 示例: `躯干晃动率 = 波动(数值=躯干垂直角, 窗口=5帧)` (场景：二头弯举-判定躯干是否借力晃动)
- **阈值**:
  - 逻辑: `数值 > 或 < 设定值`
  - 示例: `抬肘达标 = 阈值(数值=大臂垂直角, 设定=>80°)` (场景：侧平举-判定抬肘高度是否达标)

## 虚拟计算点池配置：
- **类型**：原生骨骼点 | 虚拟计算点
- **原生骨骼点**：
  - ID：模型原始ID
  - 名称：标准骨骼名称（如 `nose`, `left_shoulder`）
- **虚拟计算点**：
  - ID：自定义唯一ID (>100)
  - 名称：自定义名称 (如 `髋部中点`)
  - *算法*：引用 [核心算子枚举-几何构建类]
  - *输入*：依赖的点ID列表
  - *参数*：算法所需的额外参数

## 指标池配置：
- **说明**：统一通过核心算子定义所有需要计算的物理量，供下游模块复用。
- **基本属性**：
  - *ID*：指标唯一标识 (如 `膝关节角度`, `髋高比`)
  - *名称*：显示名称 (如 `膝关节角度`)
  - *算法*：引用 [核心算子枚举-数值度量类/逻辑判定类]
  - *输入*：依赖的点ID 或 其他指标ID
  - *参数*：算法参数
- **配置示例**：
  - ID: `下蹲深度`
  - 算法: `压缩率`
  - 输入: `当前=髋膝垂直距离, 基准=站立基准高度`
  - 参数: `无`

## 动态基准池配置（可选）：
- **说明**：定义用于计算压缩率或相对位置的参考基准值
- **基本属性**：
  - *ID*：基准唯一标识 (如 `站立基准`)
  - *名称*：显示名称 (如 `站立基准高度`)
  - *来源指标*：引用 [指标配置] 中的ID (如 `髋膝垂直距离`)
- **采集逻辑**：
  - 算法：`智能基准校准`
  - 机制：自动在`准备`状态下采集最大值，并应用阻尼抗噪和微量衰减
  - 参数：
    - 阻尼：默认 0.05 (抗噪)
    - 衰减：默认 0.9995 (防止虚高)

## 状态机配置：
- **准备**：
  - 说明：动作就绪状态，**允许采集动态基准值**
  - 逻辑：系统初始化或从复位状态判定稳定后进入
  - 属性：基准校准开关
  - 示例：`进入: 复位状态保持 > 0.5s`
- **向心**：
  - 说明：动作开始发力或下放的阶段
  - 定义：触发条件（引用指标ID + 运算器 + 阈值）
  - 示例：`下蹲深度 ≤ 85%` (深蹲开始)
- **进入纠错区间（可选）**：
  - 说明：进入核心判定区域，开始激活“过程修正”或“达成即过”逻辑
  - 定义：触发条件（引用指标ID + 运算器 + 阈值）
  - 示例：`下蹲深度 ≤ 30%` (进入深蹲底部)
- **离开纠错区间（可选）**：
  - 说明：离开核心判定区域，锁定“过程修正”结果
  - 定义：触发条件（引用指标ID + 运算器 + 阈值）
  - 示例：`下蹲深度 ＞ 30%` (离开深蹲底部)
- **离心**：
  - 说明：动作还原或起身的运动过程
  - 定义：触发条件（引用指标ID + 运算器 + 阈值）
  - 示例：`下蹲深度 ＞ 30%` (开始起身)
- **复位**：
  - 说明：动作结束后的过渡状态，**暂停基准采集**，防止动作惯性干扰
  - 定义：触发条件（引用指标ID + 运算器 + 阈值）
  - 示例：`进入: 下蹲深度 ≥ 92% (动作完成)`
- **保持（可选）**：
  - 说明：动作过程中的静力保持阶段
  - 定义：进入条件 + 退出条件 + 最小保持时长
  - 示例：`进入: 下蹲深度 < 15%; 退出: 下蹲深度 > 20%; 时长: 1s`
- **超时熔断（可选）**：
  - 说明：异常状态下的强制复位机制
  - 定义：超时阈值（秒） + 状态校验指标
  - 逻辑：若 (当前状态时长 > 超时阈值) 且 (指标满足复位条件) 则 强制转为准备态
  - 示例：`时长 > 6s AND 下蹲深度 > 75%` (深蹲卡死复位)

## 单双侧定义：
- 说明：定义左右肢体检测结果如何合并为最终判定结果
- **整体**：
  - 逻辑：`指标值不达标`
  - 说明：针对躯干、双肢协同或单一整体指标的判定，不区分左右侧。
  - 示例：`躯干侧倾 OR 深蹲下蹲幅度不足`
- **触发模式**：
  - *任一侧*：
    - 逻辑：`左侧不达标 OR 右侧不达标`
    - 说明：只要有一侧不达标，即触发错误（严判，适用于安全类指标）
    - 示例：`左膝内扣 OR 右膝内扣 → 触发错误`
  - *双侧同时*：
    - 逻辑：`左侧不达标 AND 右侧不达标`
    - 说明：两侧同时不达标时，才触发错误（宽判）
- **解除模式**：
  - *双侧达标*：
    - 逻辑：`左侧达标 AND 右侧达标`
    - 说明：必须两侧都达标，错误才解除（严进严出，推荐）
    - 示例：`左膝正常 AND 右膝正常 → 解除错误`
  - *任一侧达标*：
    - 逻辑：`左侧达标 OR 右侧达标`
    - 说明：只要有一侧达标，错误即解除（宽进宽出）

## 错误判定配置：
- **说明**：定义动作评价的时间维度判定逻辑
- **枚举**：一票否决 | 达成即过 | 过程修正 | 实时跟随
- **一票否决**：
  - 逻辑：`最终结果 = 合格 IF (动作全程 无 任何帧不达标)`
  - 触发：初始默认为合格，一旦检测到错误（满足防抖），即刻锁定为不合格。
  - 解除：本轮不可解除，需等待下一轮动作重置。
  - 场景：动作标准类，需要全程保持正确的动作
  - 示例：小臂不垂直（推肩）；耸肩（推肩）
- **达成即过**：
  - 逻辑：`最终结果 = 合格 IF (纠错区间内 存在 任意帧达标)`
  - 触发：初始默认为不合格，只要在关键区间（如最低点）达标一次，即刻锁定为合格。
  - 解除：达标即解除错误，保持合格直至本轮结束。
  - 场景：动作幅度类，需要特定关键帧区间动作幅度达标的动作
  - 示例：下蹲深度不足（深蹲）、抬肘高度不足（侧平举）
- **过程修正**：
  - 逻辑：`最终结果 = 合格 IF (区间内已修正 AND 修正后保持至离开区间)`
  - 触发：进入纠错区间时若不达标，判为不合格。
  - 解除：必须在区间内修正动作（达标），且后续保持正确姿态直到离开纠错区间。
  - 场景：动作标准类，需要特定关键帧区间保持正确动作
  - 示例：膝内扣（深蹲）；膝内扣（弓步）
- **实时跟随**：
  - 逻辑：`最终结果 = 当前帧状态` (无状态记忆)
  - 触发/解除：所见即所得，状态随每一帧检测结果实时跳变。
  - 场景：辅助参考线、非关键性的软性提示
  - 示例：脊柱曲率实时显示、重心投影点跟随

## 错误项配置架构：
- **说明**：定义单个错误项的完整配置结构，包含识别逻辑与交互反馈。

### 1. 基本信息
- **标识**：
  - *ID*：唯一标识符 (如 `valgus`)
  - *名称*：显示名称 (如 `膝内扣`)
  - *描述*：详细说明 (如 `检测下蹲过程中双膝是否内扣`)
- **属性**：
  - *优先级*：P0 (阻断) | P1 (核心) | P2 (优化)
  - *算法分类*：标准指标型 | 动力链型 | 轨迹拟合型 | 节奏速率型 | 姿态相似型

### 2. 核心算法配置 (多选一)
#### 类型A：标准指标型
- **适用场景**：基于单帧或短时序的几何指标判定（角度、距离、比例等）。
- **属性定义**：
  - *目标指标*：引用 [指标配置] ID (支持单个或多个)。
  - *阈值*：数值比较 (如 `> 80`, `< 0.1`)，与指标一一对应。
  - *组合逻辑*：`AND` (同时满足) | `OR` (任一满足) (仅多指标时有效)。
  - *容忍度*：定义不同难度等级下的判定阈值。
    - 格式：`严格 [数值] / 标准 [数值] / 宽松 [数值]`
- **配置示例1**：侧平举-抬肘高度 (单指标)
  - 目标指标：`大臂垂直角`
  - 阈值：`> 80°`
  - 容忍度：`严格 > 85° / 标准 > 80° / 宽松 > 70°`
- **配置示例2**：站姿推举-腰椎反弓 (复合指标)
  - 目标指标：`[躯干后仰角, 髋部前移量]`
  - 阈值：`[> 15°, > 0.1]`
  - 组合逻辑：`AND` (同时满足后仰且顶髋，判定为反弓代偿)
  - 容忍度：`严格 [> 10°, > 0.05] / 标准 [> 15°, > 0.1] / 宽松 [> 20°, > 0.15]`

#### 类型B：动力链型
- **适用场景**：复合动作协同关系。
- **枚举**：同步型 | 顺序型

**B1. 同步型**
- **场景**：多关节协同运动 (如：深蹲/硬拉的膝髋协同、跑步的手脚协同、早安式深蹲检测)。
- **属性定义**：
  - *基准指标*：引用 [指标配置] ID (自变量)。
  - *跟随指标*：引用 [指标配置] ID (因变量)。
  - *拟合公式*：`| 跟随 - (基准 * 系数 + 偏移) | ≤ 阈值`
  - *容忍阈值*：允许跟随指标偏离理论计算值的最大误差范围。
- **配置示例1**：硬拉-膝髋同步 (1:1 协同)
  - 基准指标：`髋伸直角`
  - 跟随指标：`膝伸直角`
  - 拟合公式：`系数=1.0, 偏移=0`
  - 容忍阈值：`≤ 15°`
- **配置示例2**：侧平举-肘部锁定 (静态稳定)
  - 基准指标：`肩外展角` (自变量变化)
  - 跟随指标：`肘屈曲角` (因变量保持)
  - 拟合公式：`系数=0, 偏移=150` (保持150度微屈)
  - 容忍阈值：`≤ 10°`

**B2. 顺序型**
- **场景**：能量传递/动力链动作 (如：划船[腿->手]、推举[腿->手]、高翻[髋->手])。
- **属性定义**：
  - *触发指标*：引用 [指标配置] ID (下游动作)。
  - *触发阈值*：激活检测的临界值 (如：肘屈曲 < 120°)。
  - *前置指标*：引用 [指标配置] ID (上游动作)。
  - *达标阈值*：前置指标必须已满足的条件 (如：膝伸直 > 160°)。
  - *时间窗口*：允许触发指标与前置指标在短时间内并发存在的容错窗口 (秒)。
- **配置示例1**：划船-过早拉臂 (上肢抢跑)
  - 触发指标：`肘屈曲角` (触发阈值 `< 120°`)
  - 前置指标：`膝伸直角` (达标阈值 `> 160°`)
  - 时间窗口：`0.1s`
- **配置示例2**：借力推举-过早推肩 (下肢驱动)
  - 触发指标：`肘伸直角` (触发阈值 `> 90°`，即开始推)
  - 前置指标：`膝伸直角` (达标阈值 `> 170°`，即腿已蹬直)
  - 时间窗口：`0.15s`

#### 类型C：轨迹拟合型
- **适用场景**：时序路径分析。
- **枚举**：直线型 | 弧线型

**C1. 直线型**
- **场景**：要求轨迹保持直线的动作（如：深蹲重心垂直移动、卧推推起轨迹）。
- **属性定义**：
  - *追踪点*：引用 [虚拟计算点] ID (需检测轨迹的动点)。
  - *参考点*：引用 [虚拟计算点] ID (相对静止的参考点，用于构建坐标系)。
  - *理想轴线*：`垂直` | `水平` | `自定义向量` (如：动作起点至终点的连线)。
  - *动态基准*：引用 [动态基准配置] ID (用于计算相对偏差的参考长度)。
  - *偏差阈值*：单帧允许的最大偏离比例（偏离距离 / 动态基准）。
  - *拟合度标准*：定义严格、标准、宽松三档的拟合度百分比阈值。
- **配置示例1**：深蹲-重心垂直度 (垂直稳定)
  - 追踪点：`髋部中点`
  - 参考点：`脚底中心` (地面投影)
  - 理想轴线：`垂直`
  - 动态基准：`躯干长度基准`
  - 偏差阈值：`< 0.1` (即偏离不超过 10% 躯干长度)
  - 拟合度标准：`严格 > 90% / 标准 > 80% / 宽松 > 70%`
- **配置示例2**：卧推-推起轨迹 (斜向向量)
  - 追踪点：`腕关节`
  - 参考点：`肩关节`
  - 理想轴线：`自定义向量` (起点=胸部落点, 终点=肩部上方)
  - 动态基准：`最大臂长基准`
  - 偏差阈值：`< 0.05` (即偏离不超过 5% 臂长)
  - 拟合度标准：`严格 > 95% / 标准 > 90% / 宽松 > 85%`

**C2. 弧线型**
- **场景**：以关节为轴心的旋转动作（如：侧平举、哑铃飞鸟）。
- **属性定义**：
  - *追踪点*：引用 [虚拟计算点] ID (做圆周运动的动点)。
  - *圆心点*：引用 [虚拟计算点] ID (旋转轴心点)。
  - *基准半径*：引用 [动态基准配置] ID (理想的旋转半径长度)。
  - *偏差阈值*：单帧允许的半径偏差比例（|实时半径 - 基准半径| / 基准半径）。
  - *拟合度标准*：定义严格、标准、宽松三档的拟合度百分比阈值。
- **配置示例1**：侧平举-手臂伸展度 (防止屈肘偷懒)
  - 追踪点：`腕关节`
  - 圆心点：`肩关节`
  - 基准半径：`最大臂长基准`
  - 偏差阈值：`< 0.15` (即半径缩短不超过 15%)
  - 拟合度标准：`严格 > 90% / 标准 > 80% / 宽松 > 70%`
- **配置示例2**：哑铃飞鸟-抱桶定型 (力臂保持)
  - 追踪点：`腕关节`
  - 圆心点：`肩关节`
  - 基准半径：`准备态臂长基准` (微屈固定)
  - 偏差阈值：`< 0.1` (即半径波动不超过 10%)
  - 拟合度标准：`严格 > 95% / 标准 > 85% / 宽松 > 75%`

#### 类型D：节奏速率型
- **适用场景**：动作节奏控制（TUT训练）、爆发力检测。
- **属性定义**：
  - *目标指标*：引用 [指标配置] ID。
  - *分析维度*：时长 | 速度 | 加速度。
  - *监测阶段*：指定检测的时间窗口 (向心 | 离心 | 全程)。
  - *阈值范围*：允许的数值区间 [最小值, 最大值]。
- **配置示例1**：深蹲-离心慢放 (TUT)
  - 目标指标：`下蹲深度`
  - 分析维度：`时长`
  - 监测阶段：`离心` (下放阶段)
  - 阈值范围：`> 3.0s`
- **配置示例2**：高翻-爆发力 (Power)
  - 目标指标：`髋部高度` (关注上升速度)
  - 分析维度：`速度`
  - 监测阶段：`向心`
  - 阈值范围：`> 1.5` (m/s)

#### 类型E：姿态相似型
- **适用场景**：难以用几何规则描述的复杂静态姿态（如瑜伽、普拉提）。
- **属性定义**：
  - *比对指标集*：引用 [指标配置] ID列表 (定义关注哪些物理量)。
  - *参考模板*：引用已录制的标准动作数据ID。
  - *比对维度*：
    - **形态**：输入必须为 [角度类] 指标，关注几何形状。
    - **位置**：输入必须为 [距离/坐标类] 指标，关注空间位置。
  - *达标阈值*：相似度得分下限 (0.0 - 1.0，越接近1.0越标准)。
- **配置示例1**：瑜伽-树式 (静态平衡)
  - 比对指标集：`[左肩展角, 右肩展角, 髋角...]`
  - 参考模板：`yoga_tree_std`
  - 比对维度：`形态`
  - 达标阈值：`> 0.85`
- **配置示例2**：普拉提-百次拍击 (核心控制)
  - 比对指标集：`[头部高度, 肩部高度, 腿部高度...]`
  - 参考模板：`pilates_hundred_std`
  - 比对维度：`位置`
  - 达标阈值：`> 0.90`

### 3. 判定策略配置
- **说明**：定义从“指标计算”到“最终判定”的数据处理流。
- **检测时机**：
  - *枚举*：自动 (默认) | 全程 | 向心 | 离心 | 静力保持 | 纠错区间
  - *说明*：指定该错误项的激活时间窗口。
  - *逻辑映射 (自动模式)*：
    - 若判定模式为 `过程修正` / `达成即过` -> 自动锁定为 `纠错区间` (依赖状态机事件)。
    - 若判定模式为 `一票否决` / `实时跟随` -> 自动锁定为 `全程`。
  - *约束*：
    - 仅在 `一票否决` / `实时跟随` 模式下，允许手动指定 `向心` / `离心` 等物理阶段（如：轨迹拟合仅检测向心）。
    - 禁止在 `过程修正` 模式下手动指定物理阶段，否则会导致逻辑截断。
- **逻辑关系**：
  - *枚举*：OR (默认) | AND
  - *说明*：当核心算法配置了多个指标时，定义其组合触发逻辑（OR=任一触发即报错）。
- **单双侧逻辑**：
  - *枚举*：整体 (默认) | 任一侧 | 双侧同时 | 双侧达标 | 任一侧达标
  - *说明*：选择适用的左右肢体合并逻辑（详见全局定义）。
- **时序防抖**：
  - *触发帧数*：连续 N 帧满足条件才触发错误（默认 3帧）。
  - *释放帧数*：连续 N 帧不满足条件才解除错误（默认 3帧）。
- **判定模式**：
  - *枚举*：实时跟随 (默认) | 一票否决 | 达成即过 | 过程修正
  - *说明*：选择适用的时间维度判定逻辑（详见全局定义）。
  - *特殊说明*：对于`轨迹拟合型`，最终考核由拟合度标准决定，此模式主要控制交互反馈的锁定与解除逻辑。

### 4. 交互反馈配置
- **说明**：定义该错误触发时的视觉反馈。
- **交互组名称**：
  - *绑定条件*：关联的错误项ID（如 `valgus`）
  - *状态：正确*：
    - *元素名称*：
      - 类型：`对勾` | `圆点` | `线` | `提示词`
      - 位置/定义：引用 [虚拟计算点] ID | 起点 -> 终点
      - 样式：颜色（如绿色）、实心/虚线
      - 内容：（仅提示词有效）
  - *状态：错误*：
    - *元素名称*：
      - 类型：`箭头` | `参考线` | `目标线` | `圆点` | `提示词` | `轨迹线`
      - 定义：
        - 几何：引用 [虚拟计算点] ID (起点) -> 引用 [虚拟计算点] ID (终点) | 起点 + 方向
        - 轨迹：引用 [虚拟计算点] ID (自动绘制历史路径)
      - 方向：`垂直` | `水平` | `向外` | `向内` | `指向目标` | `自定义向量`
      - 样式：颜色（如红色）、实心/虚线
      - 动效：`弹性` | `呼吸` | `流动`
      - 内容：（仅提示词有效）
  - *箭头生成逻辑（参数详解）*：
    - *锚点*：`起点` (箭头发出) | `终点` (箭头指向)
    - *方向逻辑*：
      - *指向目标*：连接起点与终点。
      - *轴向指引*：从锚点沿 `垂直` 或 `水平` 方向延伸。
      - *发散/汇聚*：相对于参考中心 `向外` 或 `向内`。
